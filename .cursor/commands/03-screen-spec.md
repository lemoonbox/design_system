# /screen-spec {screen-id}

## 목적
단일 화면의 기능명세서를 작성한다.
`service-plan.md`와 `ia.md`를 기준으로 필수 데이터와 기능이 누락되지 않도록 한다.

## 실행 전 읽기
1. `projects/{nn}.{project-name}/context/service-plan.md` — 핵심 기능, 사용자 맥락 확인
2. `projects/{nn}.{project-name}/context/ia.md` — 해당 화면의 위치, 진입 조건, 이동 조건 확인

## 작성 원칙
- ia.md의 이동 조건을 그대로 반영한다. 임의로 변경하지 않는다
- service-plan의 필수 기능이 이 화면에 해당된다면 반드시 포함한다
- 데이터 필드는 빠짐없이 나열한다. 불확실하면 사용자에게 확인한다
- 엣지케이스는 ia.md의 예외 플로우를 참고한다

## 작성 전 확인 (사용자에게)
- 이 화면에서 반드시 보여야 하는 데이터가 있는가?
- 이 화면에서 수행할 수 있는 모든 액션이 맞는가?
- 놓친 상태(state)가 있는가?

---

## 출력: screen-specs/{id}-{name}.md

```markdown
# {ID} — {화면명}

## 개요
- 목적: (사용자가 이 화면에서 달성하는 것, 한 문장)
- 진입 경로: (ia.md 기준)
- 이탈 경로: (ia.md 기준, 조건 포함)

## 레이아웃 구조
- 네비게이션: (상단 바 / 하단 탭 / 백버튼 / 없음)
- 헤더: (있음 — 내용 명시 / 없음)
- 주요 영역: (목록 / 폼 / 단일 콘텐츠 / 뷰어 / 혼합)
- 하단 고정 영역: (있음 — 내용 명시 / 없음)

## 데이터 필드
| 필드명 | 타입 | 필수 여부 | 표시 방식 | 비고 |
|--------|------|----------|----------|------|
| ...    | text / number / date / select / boolean | Y/N | 텍스트 / 배지 / 아이콘 | ... |

## 액션
| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 저장 | 버튼 탭 | L1으로 이동 | 필수 필드 완료 시 |
| 저장 | 버튼 탭 | 인라인 오류 표시 | 필수 필드 미완료 시 |

## 상태 정의
| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 데이터 로드 완료 | 정상 렌더 |
| 로딩 | 데이터 조회 중 | 스켈레톤 또는 스피너 |
| 빈 상태 | 데이터 없음 | 빈 상태 메시지 + 안내 액션 |
| 오류 | 네트워크 실패 등 | 오류 메시지 + 재시도 버튼 |
| 편집 중 | 사용자 입력 중 | (해당되는 경우만) |

## 엣지케이스
- (ia.md 예외 플로우 중 이 화면에 해당하는 것)
- (데이터 이상값, 권한 없음, 타임아웃 등)

## 패턴 후보
(이 화면에서 다른 화면과 공통으로 쓸 수 있는 UI 조각 메모)
- 예: "데이터 행 구조가 G1과 동일"
```

저장 위치: `projects/{nn}.{project-name}/context/screen-specs/{id}-{name}.md`

## 작성 순서
ia.md의 화면 목록 순서대로 진행한다.
한 화면 완료 후 다음 화면으로 넘어간다.
전체 화면 목록이 필요하면 `/ia-flow` 결과를 참조한다.
