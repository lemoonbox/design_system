# /visual-direction

## 목적
`design-system.md`(토큰 수치)와 별도로, 이 프로젝트가 어떤 **시각적 성격**을 가질지 정의한다.
AI가 05~09 커맨드 실행 시 "어떻게 보여야 하는가"의 기준으로 사용한다.

> ⚠️ **실행 전제**
> `/design-system` 커맨드가 완료되어 `design-system.md`가 존재해야 한다.
> 이 커맨드는 foundation 값을 새로 정의하지 않는다 — 비주얼 방향만 결정한다.

---

## 실행 전 읽기
1. `projects/{nn}.{project-name}/design-system/design-system.md` — 확정된 토큰 값 파악
2. `projects/{nn}.{project-name}/context/service-plan.md` — 서비스 성격, 사용자 맥락
3. `projects/{nn}.{project-name}/context/foundation/core-visual-tokens.json` — 팔레트 옵션 파악 (방향 논의용)

## 인터뷰 진행 방식
항목별로 **한 번에 하나씩** 질문한다. 모든 항목을 한꺼번에 묻지 않는다.
각 답변 후 AI가 내용을 요약·확인하고 다음 항목으로 넘어간다.
사용자가 "모르겠다" 또는 "AI가 결정해"라고 하면 service-plan.md 맥락을 근거로 제안하고 확인을 받는다.

## 인터뷰 항목

### Q1. 비주얼 톤
이 앱의 전반적인 느낌은?
- (A) 전문·진중 — 병원·의료·물류 등 신뢰가 먼저인 도구
- (B) 명료·효율 — 빠른 작업, 군더더기 없는 UI
- (C) 친근·활동적 — 소비자 앱, 컬러풀, 따뜻한 느낌
- (D) 직접 서술

### Q2. 색상 사용 철학
색상을 어떻게 쓸 것인가?
- (A) 최소화 — 강조는 한 컬러만, 나머지는 모노톤
- (B) 역할 분리 — primary / accent / 상태 컬러 각각 명확한 역할
- (C) 풍부하게 — 화면마다 색상 포인트, 카테고리별 컬러 구분

### Q3. 레이아웃 밀도
정보를 얼마나 빽빽하게 표시할 것인가?
- (A) 여유롭게 — 여백 넓게, 한 화면에 핵심 정보만
- (B) 중간 — 스크롤 최소화하면서 적당한 여백
- (C) 조밀하게 — 한 화면에 많은 정보, 효율 우선

### Q4. 카드 & 컨테이너 스타일
UI 요소의 외곽선·깊이감은?
- (A) 플랫 — 선으로 구분, 그림자 최소
- (B) 소프트 — 그림자 있고 라운딩 넉넉, 부드러운 느낌
- (C) 구조적 — 테두리 선명, 섹션 구분 명확

### Q5. 강조 방식
중요한 정보·행동을 어떻게 강조할 것인가?
- (A) 컬러만 — primary 컬러로 시선 유도
- (B) 크기 + 컬러 — 타이포 크기 차이와 컬러 병행
- (C) 배경 + 컨테이너 — 강조 영역에 배경색 or 카드 적용

### Q6. 추가 방향 (선택)
위에서 다루지 못한 디자인 결정사항이 있으면 자유롭게 입력.
예: 특정 색상 사용 금지, 아이콘 스타일, 특이 사용 환경 등

### Q7. 화면별 디자인 개성
콘텐츠 영역의 디자인 자유도는?
- (A) 높음 — 화면마다 고유한 레이아웃과 시각적 실험 환영
- (B) 중간 — 기본 구조는 공유하되 배치·강조·시각 처리에서 차별화
- (C) 낮음 — 대부분의 화면이 유사한 카드 기반 구조로 통일

## Surface 실효성 자동 진단 (Q4 답변 후 실행)

인터뷰 Q4 응답을 받은 직후, 다음 절차를 **자동으로** 수행한다.

### 진단 절차
1. `design-system.md`에서 background·surface·surface-raised fill 토큰의 Hex 값을 읽는다.
2. 각 Hex를 sRGB 상대 휘도(relative luminance)로 변환한다.
   - 공식: L = 0.2126×R' + 0.7152×G' + 0.0722×B' (R' = linearize(R/255))
3. 인접 계층 간 휘도 차이(ΔL)를 계산한다:
   - ΔL₁ = |L(surface) − L(background)|
   - ΔL₂ = |L(surface-raised) − L(surface)|
4. 이펙트 토큰(shadow)의 alpha 합산값을 보정 계수로 반영한다.

### 판정 기준
- **ΔL ≥ 0.04 (4%)**: 충분 — 조치 불요
- **0.02 ≤ ΔL < 0.04**: 경고 — shadow alpha 합산 ≥ 0.15이면 허용, 미만이면 토큰 조정 권장
- **ΔL < 0.02**: 위험 — 반드시 토큰 조정을 제안

### 제안 형식
판정이 "경고" 또는 "위험"일 때, 다음과 같이 사용자에게 제안한다:

> ⚠️ **Surface 계층 실효성 진단 결과**
>
> | 계층 | 현재 토큰 | Hex | 휘도 |
> |------|----------|-----|------|
> | background | {토큰} | {hex} | {L값} |
> | surface | {토큰} | {hex} | {L값} |
> | surface-raised | {토큰} | {hex} | {L값} |
>
> - ΔL₁ (background↔surface): {값} → {충분/경고/위험}
> - ΔL₂ (surface↔surface-raised): {값} → {충분/경고/위험}
>
> **권장 조정:**
> - (예) background를 gray.100(#f5f5f5)으로 내려 ΔL₁을 {새 값}으로 확보
> - (예) surface를 white(#ffffff)로 올려 ΔL₂를 {새 값}으로 확보

사용자가 조정을 승인하면 visual-direction.mdc 출력에 반영하고, design-system.md 수정도 함께 안내한다.

## Surface 계층 실효성

> Surface 실효성 자동 진단 결과를 기록한다. 조정이 있었으면 조정 전→후를 명시한다.

| 계층 쌍 | 토큰 A → 토큰 B | ΔL | shadow 보상 | 판정 |
|---------|----------------|-----|------------|------|
| 배경 ↔ 카드 | {토큰} → {토큰} | {값} | {shadow 토큰 alpha 합} | 충분/보상으로 충분/조정됨 |
| 카드 ↔ 입력면 | {토큰} → {토큰} | {값} | — | 충분/조정됨 |

- 최소 대비 기준: ΔL ≥ 0.04 또는 shadow alpha 합 ≥ 0.15로 보상
- {조정 항목이 있으면 "조정 전 → 조정 후" 기록}

## 색상 분배 규칙

> Q1(비주얼 톤) + Q2(색상 철학) 답변으로부터 AI가 자동 도출한다. 사용자에게 별도 질문하지 않는다.

### 도출 원리
- Q1 답변의 서비스 성격과 Q2 답변의 색상 범위를 조합한다.
- 목표: "CTA 버튼만 유일한 색상"인 상태를 벗어나되, 비주얼 톤이 허용하는 범위 안에서 기능적 색상 포인트를 배치한다.

### 색상이 허용되는 기능적 위치
{Q1+Q2 기반으로 아래 목록에서 해당하는 항목을 선택·서술}

| 위치 | 허용 색상 | 역할 | 면적 기준 |
|------|----------|------|----------|
| 카드 좌측 상태 바 | success / warning / error / primary | 상태 즉시 인지 | 너비 4px, 카드 좌측 모서리 |
| 섹션 헤더 좌측 바 | primary 또는 accent | 섹션 시작점 앵커 | 너비 3~4px, 높이 = 제목 행 |
| 스텝퍼 노드 | primary(현재) / success(완료) / gray(미완) | 진행 상태 | 원형 24~32px |
| 카테고리·상태 배지 | 상태별 subtle 배경 + 아이콘 + 문구 | 분류·상태 표시 | 높이 24~28px 칩 |
| 아이콘 강조 | accent 또는 상태색 | 행 시작점 시각 앵커 | 아이콘 크기 내 |

### 색상이 허용되지 않는 위치
- 장식 목적 배경 컬러 블록
- 카드 전체 fill을 상태색으로 칠하기
- 구분선·여백을 색으로 채우기
- 시각적 "풍성함"만을 위한 무목적 accent

### 색상 면적 예산
- primary fill 면: 화면당 CTA 1곳 + 소면적 강조 최대 3곳 이내
- 상태색(success/warning/error): 해당 상태가 존재할 때만, 아이콘+문구 병행 필수
- accent: 정보성 링크·거리 등 보조 역할 한정
