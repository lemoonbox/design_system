# 디자인 명세 — L4 이송 병원 추천·조율

## 화면 개요
- **목적**: 추천 병원을 검토·다중 이송 신청·승인 상태를 확인한 뒤 한 곳만 출발 확정하고, 이송 중·도착 후에도 같은 화면 흐름으로 케이스를 마무리한다.
- **디바이스**: 390 × 844 모바일
- **사용자 맥락**: G1·L2에서 진입한 직후, 또는 출발 확정 후 이송 중·병원 도착 직전까지. 조도 변화·장갑 착용 환경에서 카드 단위로 상태를 빠르게 스캔해야 한다.
- **핵심 행동**: **승인된 병원에서 출발 확정(M1)** 또는 **이송 중 「병원 도착」(M4)** — 모드에 따라 주 행동이 바뀐다.

## 시각적 의도

> 카드 목록이 곧 업무 보드다. 병원별 가용·거리·과·신청 상태가 한눈에 구분되고, 위험한 오인을 막기 위해 상태는 색만이 아니라 아이콘·짧은 문구로 고정된다.

- **분위기**: 진중한 전문가 도구. 장식 그래픽 없이 **병원 카드 스택**과 헤더 모드 배지로 맥락이 즉시 읽힌다.
- **시각적 초점 (focal point)**: 스크롤 본문의 **첫 번째·가장 상태가 액션 가능한 병원 카드**(병원명 + 여유 병상 + 상태 배지). 이송 중 모드에서는 **하단 「병원 도착」primary CTA**가 행동 초점을 고정한다.
- **색상 톤**: 페이지 `color.gray.25`, 카드 `color.gray.50` + `effect.shadow.sm`, 칩·필드형 정보는 `color.white`(surface-raised)로 얹어 **배경 → 카드 → 요소 → 텍스트** 4단계를 유지한다. 거리·보조 링크는 **accent**(`color.blue.600`)로 primary(바이올렛)와 역할을 분리한다.
- **밀도**: 중간. 카드 내 버튼·배지·행 탭 영역은 최소 **44px** 높이(또는 동등 히트 영역)를 확보한다.
- **디자인 핵심**: 카드에만 `effect.shadow.sm`, 하단 고정 바(이송 중)는 `effect.shadow.md`로 본문과 분리한다. **shadow와 stroke를 동시에 쓰지 않는다**(visual-direction 소프트 elevation).

## Visual Recipe

화면 최상단은 PAT-001·PAT-002로 케이스·동기화 맥락을 L1·L2와 동일 리듬으로 고정한다. 그 아래 **내비게이션 행**은 뒤로·제목 **「이송·병원」**·우측 **모드 배지**(「추천·신청」/「이송 중」)가 한 줄에서 읽힌다. PAT-003 **축약 스텝퍼**는 현재 단계가 이송 조율임을 짧게 보여 준다. 본문은 좌우 `spacing.xl` 안에서 **병원 카드**가 `spacing.lg` 간격으로 쌓이며, 각 카드는 병원명·여유 병상·거리(accent)·의료과 칩·(선택) 추천 사유 한 줄·상태 배지·카드 하단 **이송 신청 / 출발** 버튼으로 끝난다. **추천·신청** 모드에서는 주 행동이 카드 내부에 있으므로 하단 고정 바는 두지 않고 리스트 하단 여백으로 마무리한다. **이송 중** 모드로 바뀌면 헤더·카드 비활성/취소 표기가 바뀌고, PAT-004로 **「병원 도착」** 단일 primary가 `effect.shadow.md`로 화면 하단을 고정한다. M1·M2·M4는 PAT-005 오버레이로 동일 톤을 유지한다.

### 강조 검증

- focal point: **추천·신청** — 첫 액션 가능 카드의 병원명+상태 배지 / **이송 중** — 하단 「병원 도착」primary
- primary 사용: 카드 **출발** 버튼, 하단 **병원 도착**, (선택) 단일 강조 링크 외 남용 금지

- 터치 가능 요소: 뒤로·모드 배지·칩·「이송 신청」「출발」「병원 도착」— 높이 **≥44px** 또는 동등 히트 영역
