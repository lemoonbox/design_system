# 디자인 명세 — L1 음성·구급일지

## 화면 개요
- **목적**: STT·NER로 쌓인 구급 일지를 시간순으로 확인·편집하고, 다음 단계(L2)로 넘어갈 수 있게 한다.
- **디바이스**: 390 × 844 모바일
- **사용자 맥락**: 출동 직후 현장에서 말로 기록이 쌓이는 동안·이후, 장갑·조도 변화 환경에서 구간별 텍스트를 빠르게 스캔·보완할 때.
- **핵심 행동**: 하단 단일 주요 CTA로 **「증상·pre-KTAS로」** 를 눌러 L2로 이동한다.

## 시각적 의도

> 진중한 로그·타임라인 화면. 카드 단위로 구간이 끊겨 읽히고, 저신뢰 구간은 색·아이콘·문구로 편집을 유도하되 L2 진입은 막지 않는다.

- **분위기**: 의료 현장 기록 도구. 장식 없이 타임스탬프·본문·엔티티 칩의 위계가 명확하다.
- **시각적 초점 (focal point)**: **시간순 발화 카드 스택**의 최신/주목 구간(본문 STT 텍스트). 다음으로 하단 **primary CTA**가 행동을 고정한다.
- **색상 톤**: 페이지 배경 gray.25, 카드는 surface(gray.50) + shadow.sm. primary는 CTA만. 신뢰도·경고는 success/warning/error를 **배지+아이콘+짧은 문구**와 병행한다.
- **밀도**: 중간. 행·버튼·배지 탭 영역 최소 **44px** 확보.
- **디자인 핵심**: 카드에만 shadow, 카드와 페이지 배경의 fill 대비로 계층을 만든다(shadow와 stroke 동시 사용 금지).

## Visual Recipe

화면 상단은 PAT-001·PAT-002로 케이스·동기화 맥락을 G1·L2·L4와 동일 리듬으로 고정한다. 그 아래 **내비게이션 행**은 흰 표면 위에 뒤로(chevron)·화면 제목 **「구급 일지」**·우측 **수집 상태**(예: mic + 「수집 중」)가 한 줄로 읽힌다. 본문은 좌우 spacing.xl 안전 영역 안에서 세로 스크롤되며, 각 **발화 구간 카드**는 상단에 구간 시각(caption), 그 아래 신뢰도 배지·편집 액세스, STT 본문(body), NER 칩 행 순으로 쌓인다. 저신뢰 구간은 warning 톤 배지와 짧은 안내 문구로 편집 권장만 표시한다. 하단은 PAT-004로 **「증상·pre-KTAS로」** 단일 버튼이 shadow.md로 본문과 분리된다.
