# 디자인 명세 — L4 이송 시작 완료

## 화면 개요
- **목적**: M1 확정 직후 **이송이 시작되었음**을 전용 화면으로 알리고, **구급 일지(G1)로 복귀**하거나 정책에 따라 다음 케이스로 넘어간다.
- **디바이스**: 390 × 844 모바일
- **사용자 맥락**: 병원 수락·확정 직후 현장에서 한 번 더 **상태를 확인**하고 곧바로 일지 업무로 돌아가야 하는 짧은 전환 구간.
- **핵심 행동**: **「구급 일지로」**를 눌러 G1으로 이동하고(이송 중 플로팅 ON), 필요 시에만 **다음 케이스** 등 보조 행동을 선택한다.

## 시각적 의도

> 이 화면이 완성되었을 때 **어떻게 보이고 느껴져야 하는지** 서술한다.
> visual-direction.mdc의 비주얼 톤을 이 화면에 구체적으로 적용한 기술.

- **분위기**: **성공·안심**이 한눈에 드러나는 완료 화면이되, 장식적 축하보다 **도구의 확정 피드백**에 가깝게 유지한다. 정보는 **히어로 + 요약 카드 + 고정 CTA**로 끊어 읽힌다.
- **시각적 초점 (focal point)**: 상단 **성공 히어로 영역**(피드백 체크 원형 + 완료 제목)과 바로 이어지는 **최종 이송 병원명** — 시선이 위에서 아래로 **「완료 → 어디로 가는지」**로 흐른다.
- **색상 톤**: 페이지는 **background** 중립, 본문 블록은 **surface** 흰 카드 + **1px border(gray.300)**. 성공은 **success**를 **아이콘·좌측 4px 상태 바·배지** 등 소면적에만 쓰고, **주 CTA는 primary** 한 곳에 집중한다.
- **밀도**: 중간 — `spacing.xl`(16px) 패딩, 섹션 간 `spacing.3xl`(24px). 완료 화면이라 여백이 넓어도 **히어로·카드·메타 행·하단 바**로 빈 단색 면이 과반을 차지하지 않게 구성한다.
- **디자인 핵심**: **success + 문구 병행**으로 상태를 확정하고, 요약 카드는 **좌측 success 4px 바**와 **불릿/메타 행**으로 텍스트만 있는 카드를 피한다. 하단 고정 영역은 콘텐츠 위 레이어이므로 **shadow-md**만 사용(테두리와 그림자 동시 사용 금지).

## Visual Recipe

배경은 **background — gray.50 (#FAFAFA)** 이고, 스크롤 영역 상단에 **surface** 카드형 블록을 두어 **성공 아이콘(success 톤)** 과 **완료 제목(heading)** 을 세로로 쌓는다. 그 아래 **최종 이송 병원명**은 **subheading**으로 한 줄 강조하고, **확정 시각**은 아이콘+캡션 행으로 붙인다. **주증상·평가 요약**은 **이웃 카드**로 분리하고, 카드 **좌측 success 4px 세로 바**와 **불릿 body**로 정리하며, 섹션 제목 행은 (선택) **info** 아이콘으로만 풍성화한다(제목 옆 **primary 세로 바는 사용하지 않음** — 카드 좌측 바와 이중 앵커 방지). 선택 필드(안전 안내·세션 ID·동기 배너)는 **caption**·얇은 배너로만 덧붙인다. 화면 하단은 **PAT-003** 형태의 바에 **primary fill「구급 일지로」** 와 **보조 텍스트/고스트 버튼「다음 케이스」** 를 양끝 또는 세로 스택으로 배치한다.

### 강조 검증
- focal point: **성공 아이콘 + 완료 제목 + 병원명** — success·타이포 크기로 처리 — 완료 피드백이 목적.
- primary 사용: 화면 내 **주 CTA 1곳**(구급 일지로) + 소면적 강조는 success·아이콘 위주.
- 상태 표현: 성공은 **checkCircle + success 색**; 경고·오류는 **아이콘 + 문구** 병행.

### 밀도 검증
- 터치 가능 요소: 뒤로·주 CTA·보조 버튼 — 높이 **최소 44px** 권장.
- ❌ 미충족 시: 하단 바 패딩 증가 또는 버튼 세로 스택.

### 시각 밀도 검증
- 스크롤 영역 중 빈 배경 비율 추정: **~35–45%** — 히어로 카드+요약 카드+메타로 **50% 미만** 유지 목표.
- 텍스트 전용 카드: 없음 — **좌측 바·불릿·(선택) info** 로 보강.
- 반복 카드 차별화: 해당 없음(단일 요약).
- 섹션 헤더 앵커: 요약 카드 **좌측 success 4px 바** + (선택) 제목 옆 info.
- 네비 요소 무게감: **하단 primary 전폭 CTA** 로 종료감 확보.
