# 디자인 명세 — L3 이송 승인·시작

## 화면 개요
- **목적**: 일괄 신청한 병원별 승인·거절·대기 상태를 확인하고, 승인된 병원 중 1곳만 골라 이송 시작(M1)으로 이어간다.
- **디바이스**: 390 × 844 모바일
- **사용자 맥락**: L2에서 신청 제출 직후 또는 「병원 응답 보기」로 재진입한 구급 현장 — 응답 갱신을 확인하며 빠르게 출발 병원을 확정해야 하는 상황.
- **핵심 행동**: 승인 병원 1곳을 선택한 뒤 **「이송 시작」**을 눌러 다음 단계로 진행한다.

## 시각적 의도

> 이 화면이 완성되었을 때 **어떻게 보이고 느껴져야 하는지** 서술한다.
> visual-direction.mdc의 비주얼 톤을 이 화면에 구체적으로 적용한 기술.

- **분위기**: 신뢰감 있는 상태 대시보드에 가깝다. 병원별 결과가 **카드 단위**로 스캔되고, 승인/대기/거절이 **색·아이콘·좌측 상태 바**로 즉시 구분된다.
- **시각적 초점 (focal point)**: 스크롤 영역의 **병원 응답 카드 목록** — 특히 **승인 행의 라디오 선택 가능 영역**이 다음 행동(하단 CTA)으로 이어지는 시각적 축이다.
- **색상 톤**: 페이지는 **background** 중립, 카드는 **surface** + **1px border**로 구조적 구분. primary는 **선택된 승인 행·주 CTA·섹션 앵커**에 소면적 집중. success / warning / error는 **상태 배지·좌측 4px 바**에만 사용한다.
- **밀도**: 중간 — `spacing.xl`(16px) 패딩, 카드 스택 간 `spacing.3xl`(24px). 라디오 행·CTA·뒤로는 **최소 44px** 터치 타깃.
- **디자인 핵심**: L2와 동일 계열의 **병원 행(PAT-002)** 이지만 **체크 대신 승인 전용 라디오**와 **상태 배지 변형**으로 차별화. 하단 고정 바는 콘텐츠와 **레이어 분리**를 위해 **shadow-md만** 적용(테두리와 그림자 동시 사용 금지).

## Visual Recipe

배경은 **background — gray.50 (#FAFAFA)** 이고, 상단 앱 바 아래에 **갱신 시각·풀투리프레시 맥락**이 한 줄로 보인 뒤, **「병원 응답」** 섹션(좌측 **primary 3~4px 세로 바** 앵커)과 **1px border** 흰 카드 목록이 이어진다. 각 카드 좌측 **4px 상태 바**는 대기·승인·거절·만료에 따라 **warning / success / error / gray.300** 톤으로 바뀌어 **3장 이상 연속**이어도 행 단위로 즉시 구분된다. 승인 행만 **라디오(원형 선택)** 가 활성이고, 거절 행에는 **사유 본문**과 **alertTriangle** 등이 짝을 이룬다. 화면 하단 **PAT-003** 은 선택 요약(예: 병원명 한 줄)과 **primary fill「이송 시작」** 으로 시선을 마감하며, 승인 0곳·미선택 시 CTA는 **text-disabled** 톤의 비활성 표면으로 기대를 낮춘다.

### 강조 검증
- focal point: **병원 응답 카드 스택 + 승인 행 라디오** — 상태 바·배지·라디오 조합으로 처리.
- primary 사용: **이송 시작 CTA 1곳** + **선택 라디오 원** + **섹션 좌측 바** + (선택 시) **스텝 없음** → 면적 예산 내.
- 상태 표현: **아이콘 + 색 + 문구** 병행(거절·오류·대기).

### 밀도 검증
- 터치 가능 요소: 뒤로·라디오 행·CTA·보조 링크 — 높이 **≥44px** 권장.
- ❌ 미충족 시: 카드 패딩·라디오 히트 영역을 행 전체로 확장.

### 시각 밀도 검증
- 스크롤 영역 중 빈 배경 비율 추정: **낮음**(카드 다수 + 섹션 헤더 + 갱신 행) — **50% 미만 충족**.
- 텍스트 전용 카드: 거절 사유만 긴 경우 — **좌측 바·아이콘·배지**로 보강.
- 반복 카드 차별화: 동일 구조 **3+** — **좌측 4px 바 색**으로 상태 구분.
- 섹션 헤더 앵커: **있음** — 좌측 primary 3~4px 바 (+선택 list 아이콘).
- 네비 요소 무게감: **탭 없음** — 해당 없음; **라디오·CTA** 로 네비게이션 무게 확보.
