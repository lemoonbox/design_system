# 디자인 명세 — M1 이송 시작 확인

## 화면 개요
- **목적**: 승인된 **병원 1곳**에 대해 이송 출발 직전 **주증상·pre-KTAS·P-class** 맥락을 최종 확인하고, 확정하거나 취소한다.
- **디바이스**: 390 × 844 모바일(전면 **모달 오버레이** + 중앙 확인 시트)
- **사용자 맥락**: L3에서 병원을 고른 뒤 **「이송 시작」** 직전, 현장에서 실수 없이 한 번 더 맥락을 스캔해야 하는 순간.
- **핵심 행동**: 내용을 확인한 뒤 **「확인(이송 시작 확정)」**을 눌러 L4로 넘기거나, **「취소」**로 L3에 복귀한다.

## 시각적 의도

> 이 화면이 완성되었을 때 **어떻게 보이고 느껴져야 하는지** 서술한다.
> visual-direction.mdc의 비주얼 톤을 이 화면에 구체적으로 적용한 기술.

- **분위기**: 긴장은 있되 과장 없이, **한 장의 흰 시트** 안에서 맥락을 빠르게 읽고 결정할 수 있는 **도구형 확인 대화상자**. 배경(L3)은 어둡게 가려져 주의가 모달로 모인다.
- **시각적 초점 (focal point)**: 모달 본문 상단의 **병원명(강조)** 과 하단의 **primary fill 「이송 시작 확정」** 버튼 — 시선이 위→아래로 흘러 행동으로 이어지게 한다.
- **색상 톤**: 스크림은 중성 딤, 카드는 **surface 흰색**. 구조는 **1px border(gray.300)** 로 카드 외곽을 잡거나, 레이어 분리가 더 중요하면 **그림자만** 선택(동시 사용 금지). primary는 **주 CTA 1곳** + 섹션 앵커 등 소면적만.
- **밀도**: 중간 — 모달 내부 패딩 **spacing.xl(16px)**, 블록 간 **spacing.3xl(24px)**. 버튼·탭 영역 **최소 높이 44px**.
- **디자인 핵심**: L3와 겹치는 레이어이므로 시트에 **shadow-lg**로 깊이를 주고(테두리와 병행하지 않을 것), 본문은 **아이콘+라벨+본문** 행으로 빈 면을 줄인다. 오류·오프라인은 **아이콘+문구** 인라인 블록으로 상태를 분리한다.

## Visual Recipe

전체 화면 위에 **gray.900 기반 반투명 스크림**을 깔고, 가로 여백 **spacing.xl(16px)** 을 둔 **중앙 정렬 모달 카드**(최대 너비 앱 폭 대비 안전 여백 유지)를 띄운다. 카드는 **surface — white (#FFFFFF)**, **radius.lg (10px)**; 외곽은 **border — gray.300 (#D5D7DA) 1px** *또는* **shadow-lg** 중 하나만 적용해 L3와 계층을 분리한다. 상단 타이틀 행 아래에 **primary 3~4px 세로 바**로 시작하는 **「이송 맥락」** 섹션을 두고, 병원명은 **subheading**, 그 아래는 **아이콘 20px + body** 조합의 요약 행(주증상·pre-KTAS·P-class)으로 채운다. P-class는 **label 타이포의 칩**으로 압축한다. 하단은 **취소(보조)** 와 **확인(주 CTA)** 을 한 행에 두되 높이 44px 이상으로 장갑 터치에 대응한다. API 실패 시 타이틀과 버튼 사이에 **surface-raised — gray.100 (#F5F5F5)** 배경의 인라인 경고 행을 삽입한다.

### 강조 검증
- focal point: **병원명(subheading)** + **확인 버튼(primary fill)** — 확인 흐름 고정.
- primary 사용: **주 CTA 1곳** + 섹션 좌측 바·(로딩 시 아이콘) 등 소면적 — 면적 예산 준수.
- 상태 표현: 오류·오프라인은 **아이콘 + 색 + 문구** 병행.

### 밀도 검증
- 터치 가능 요소: **취소·확인** — 높이 **≥ 44px** — 충족.
- 스크림 탭: 정책에 따름(취소 동일 시 타깃 충분).

### 시각 밀도 검증
- 스크롤 영역: 모달 본문이 짧을 수 있음 → **아이콘+행 구조·섹션 바·칩**으로 빈 면 최소화.
- 텍스트 전용 카드: 해당 없음(모달 단일 블록) — 요약 행에 **아이콘** 필수.
- 반복 카드: 1건 요약 — **좌측 primary 섹션 바**로 블록 앵커.
- 섹션 헤더 앵커: **있음** — 좌측 3~4px primary 바.
- 네비 요소 무게감: **이중 버튼** — 주·보조 대비 명확, 높이 44px 이상.
