# 디자인 명세 — L2 증상·pre-KTAS

## 화면 개요
- **목적**: 일지 기반으로 제안된 증상·pre-KTAS를 확인·보완한 뒤, 병원 추천·이송(L4)으로 넘어갈 수 있게 한다.
- **디바이스**: 390 × 844 모바일
- **사용자 맥락**: 이송 전 단계에서 현장·차량 안에서 장갑을 낀 채 필수 증상·바이탈·등급을 빠르게 점검·수정할 때.
- **핵심 행동**: 필수 항목을 채우고 하단 **「병원 추천·이송」** 으로 L4로 진행한다(미충족 시 경고 확인 후 진행).

## 시각적 의도

> 진중한 응급 기록 폼. 섹션 카드로 구역을 나누고, pre-KTAS 등급 선택은 스캔 가능한 리스트·카드로 고정한다. 색은 상태·검증·주요 행동에만 쓴다.

- **분위기**: 의료 도구형 폼. 장식 없이 카드·라벨·필드 리듬으로 읽히며, 여백으로 필드 그룹을 분리한다.
- **시각적 초점 (focal point)**: 화면 하단 **PAT-004 primary 「병원 추천·이송」** 이 유일한 강한 색 면으로 행동을 고정한다. 그 직전 스크롤 흐름에서 **pre-KTAS 등급 선택 블록**이 임상 판단 앵커로 시선을 모은다.
- **색상 톤**: 페이지 배경은 gray.25, 본문 카드는 surface(gray.50) + effect.shadow.sm. 입력 칩·필드는 surface-raised(white)로 한 단계 띄운다. primary는 CTA·핵심 선택 강조에만, accent(blue.600)은 보조 링크·정보에만 제한한다. 인라인 오류는 error 색 + 아이콘 + 문구 병행.
- **밀도**: 중간. 터치 가능 행·칩·등급 카드·버튼은 최소 **44px** 높이를 확보한다.
- **디자인 핵심**: shadow로만 깊이를 주고 stroke와 동시 사용하지 않는다(visual-direction). **저장**은 PAT-004에 넣지 않고 상단 보조 액션으로 둔다(PAT-004는 단일 primary 전용).

## Visual Recipe

페이지는 밝은 회색 배경(color.gray.25) 위에 PAT-001·002로 케이스·동기화 맥락을 고정하고, 본문은 좌우 spacing.xl 안전 영역 안에서 세로 스크롤한다. 첫 머리에는 뒤로와 화면 제목(heading), 그 아래 **현재 선택된 pre-KTAS 등급 배지**가 단계 인지를 돕고, PAT-003 축약 스텝퍼로 흐름 위치를 짚어 준다. 이어지는 네 개의 카드형 섹션(필수 증상 → 필수 바이탈 → pre-KTAS·근거 → 특이사항)이 surface + shadow.sm로 본문을 끊어 읽히게 하며, 필드는 흰 입력면(surface-raised)으로 카드 배경과 대비된다. 최하단은 흰 CTA 바(effect.shadow.md)에 **병원 추천·이송** 한 줄만 두어 시선이 행동에 모인다.
