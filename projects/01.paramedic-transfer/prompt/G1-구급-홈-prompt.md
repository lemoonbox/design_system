# 디자인 명세 — G1 구급 홈

## 화면 개요
- **목적**: 진행 중 케이스의 단계와 요약을 한눈에 스캔하고, 주 행동을 음성·구급일지(L1)로 자연스럽게 연결한다.
- **디바이스**: 390 × 844 모바일
- **사용자 맥락**: 출동 후 현장·이송 중, 조도 변화가 큰 환경에서 장갑을 낀 채로 빠르게 상태를 확인할 때.
- **핵심 행동**: 하단 단일 주요 CTA로 **「음성·구급 일지」** 를 눌러 L1으로 이동한다.

## 시각적 의도

> 진중하고 스캔 가능한 전문가용 홈. 장식 없이 카드·스텝으로 구역을 나누고, 색은 상태·행동에만 쓴다.

- **분위기**: 의료 현장 도구다운 절제된 UI. 정보는 카드와 스텝 단위로 끊어 읽히며, 여백으로 호흡을 준다.
- **시각적 초점 (focal point)**: 화면 하단 **primary 단일 CTA**(음성·구급 일지)가 가장 강한 시선과 행동 유도를 맡는다. 그다음으로 **PAT-003 스텝퍼의 현재 단계**가 단계 위계로 눈에 들어온다.
- **색상 톤**: 페이지 배경은 중립(gray.25), 카드는 surface(gray.50) + shadow.sm으로 미세한 들뜸. primary는 CTA·현재 단계 강조에만, accent(blue.600)은 거리·정보 링크성 보조에만 제한적으로 사용한다.
- **밀도**: 중간. 터치 가능 행·버튼은 최소 **44px** 높이를 확보한다.
- **디자인 핵심**: shadow만으로 깊이를 주고 stroke와 혼용하지 않는다(visual-direction 소프트 elevation). 상태는 **색 + 아이콘 + 문구**를 함께 쓴다.

## Visual Recipe

페이지 전체는 아주 밝은 회색 배경(color.gray.25) 위에, 상단은 PAT-001·002로 케이스와 동기화 맥락이 고정되고, 본문은 좌우 spacing.xl 안전 영역 안에서 세로로 쌓인다. 본문 첫 줄에는 큰 화면 제목(heading 30px)으로 **「구급 홈」** 이 단계형 흐름의 앵커가 된다. 이어지는 스텝퍼는 세 단계를 한 줄에 읽을 수 있게 정렬되고, 그 아래 요약 카드가 일지·pre-KTAS·이송 상태를 짧은 줄로 정리한다. 하단은 흰 배경 위로 떠 있는 듯한 CTA 바(effect.shadow.md)가 화면을 마감하며, primary 버튼 하나만이 유일한 강한 색 면으로 남는다.

### 강조 검증
- focal point: **하단 primary CTA** — brand.600 면 + 충분한 터치 높이 — "주요 행동은 크기+컬러 병행"
- primary 사용: 화면 내 **1곳**(주요 버튼) + 스텝퍼 현재 단계 등 소면적 강조만 허용
- 상태 표현: 아이콘 + 색 + 문구 병행 — visual-direction 확정

### 밀도 검증
- 터치 가능 요소: **primary CTA** — 높이 최소 **44px** — 충족 필수
- PAT-003·요약 행이 탭 가능한 경우 — 행 높이 **≥44px** 또는 터치 영역 확장
- ❌ 미충족 시: 패딩·행 높이를 spacing.lg 이상으로 조정
