# Service Plan — 구급 이송 조율 모바일 앱

## 서비스 개요

- **목적**: 앰뷸런스 구급대원이 현장에서 환자를 평가하고, 증상에 맞는 병원으로 이송을 조율할 수 있도록 돕는다.
- **해결하는 문제**: 병원 선정·이송 조율을 **전화**로 하고, **센터 복귀 후 PC**에 기록하는 현재 방식에서는 **기억 소실**이 크고, **업무가 매우 바쁜 상황에서 수기 메모 부담**이 크다.

## 사용자

- **주요 사용자**: 앰뷸런스 구급대원
- **사용 맥락**: 환자를 만난 뒤, **현장 도착 후 처치를 시작하는 시점부터** 사용. 처치 시작과 함께 **기록이 자동으로 시작**된다고 가정한다.
- **핵심 행동** (앱 관점):
  1. 음성 기반으로 구급 일지가 자동 기록되도록 말한다 (STT·NER).
  2. 기록을 바탕으로 환자 증상·pre-KTAS를 확인·보완한다.
  3. 추천 병원을 검토하고 이송을 신청한다.
  4. 병원 승인 상태를 확인한 뒤, 승인된 병원 중 한 곳을 골라 이송을 시작한다.

## 기능 범위

### 필수 기능

- 구급대원 음성 입력 시 **STT 및 NER**로 구급 일지 **자동 기록**
- 구급 일지가 쌓이면 해당 내용을 바탕으로 **환자 증상** 및 **pre-KTAS** 평가
- 환자 증상을 바탕으로 **이송 가능 병원 추천**
- 구급대원이 **병원을 선택해 이송 신청**
- 각 병원이 이송을 승인하면 **이송 승인 상태**로 표시
- **이송 승인 상태인 병원 중 한 곳을 선택**해 **이송 시작**

### 선택 기능

- 없음 (위 항목을 이번 앱 디자인 범위의 필수로 둠)

### 제외

- **병원용 웹** (별도 제공 가정, 이번 구현·디자인 범위에서 제외)
- **센터 PC 기록 연동**, **결제**, **환자·가족용 기능**
- **기존 시스템 연동**은 앱 **화면·카피에서 언급하지 않음**
- **전화 통화와 동시에 앱을 사용하는 시나리오**는 이번 디자인에서 고려하지 않음

## 비즈니스 목표

- **목표**: **현장 기록·평가 소요 시간 단축**, **적절 병원 이송 비율 향상**
- **성공 지표** (실제 운영 시 측정 가정):
  - 구급일지 **완료까지 걸린 시간**
  - **pre-KTAS 재분류 횟수**
  - **이송 후 적합성 피드백**

## 제약 조건

- **플랫폼**: **iOS·Android** 모바일 앱
- **오프라인·네트워크 불안정**: UI·시나리오에서 **고려** (동기화·재시도·상태 표시 등 화면 수준)
- **연동 시스템**: 디자인 산출물에서 **연동을 전제로 한 설명·카피는 넣지 않음**
- **브랜드/규정**: 별도 브랜드 가이드·의료 UI 규정 **없음** (프로젝트 foundation·디자인 시스템에서 정의)

## 디자인 단계 범위

- 본 문서는 **UI 구성에 필요한 데이터·흐름**까지 정의 가능하나, **실제 구현(코딩)에 필요한 기술 스펙**까지는 요구하지 않는다.
