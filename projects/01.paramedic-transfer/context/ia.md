# IA & Service Flow — 구급 이송 조율 모바일 앱

## 화면 목록

| ID | 화면명 | 레벨 | 설명 |
|----|--------|------|------|
| G1 | 구급 홈 | Global | 진행 중 케이스 요약, 단계 진행 상태, L1·L2·L4 등으로 이동 |
| L1 | 음성·구급일지 | Local | STT/NER 자동 기록 표시, 발화 구간·텍스트 확인·보완 편집 |
| L2 | 증상·pre-KTAS | Local | 일지 기반 증상·pre-KTAS 확인, 누락 보완·재분류 |
| L4 | 이송 병원 추천·조율 | Local | 추천 병원 카드 목록. **여러 병원에 동시에 이송 신청 가능.** 카드 하단에 이송 신청, 승인 상태, 출발(이송 시작) 제공. **출발은 승인된 병원 중 1곳만** 선택. 이송 시작 후 **동일 화면(L4)** 에서 **이송 중** 상태 유지. 병원 **도착 후 케이스 완료 여부** 확인 |
| M1 | 확인 | Modal | 출발 최종 확인, 뒤로가기 시 미반영 데이터 경고 등 |
| M2 | 동기화·오류 | Modal | 오프라인·전송 실패 시 재시도, 대기 상태 안내 |
| M3 | 출동 종료 경고 | Modal | L1에서 G1 등 **상위로 이탈(뒤로가기 등)** 시, **지금 출동을 완료하는지** 경고·확인 |
| M4 | 이송·케이스 완료 확인 | Modal | **병원 도착** 후 **이번 케이스를 완료할지** 질문·확인. **‘완료’ 확인 시 [G1] 구급 홈**으로 이동 |

## IA 구조

- [G1] 구급 홈
  - [L1] 음성·구급일지
    - [M3] 출동 종료 경고
  - [L2] 증상·pre-KTAS
  - [L4] 이송 병원 추천·조율
    - [M1] 확인
    - [M2] 동기화·오류
    - [M4] 이송·케이스 완료 확인

## 네비게이션 패턴

- **타입**: 단계형(핵심 업무 순서) + 화면 스택. G1은 동일 케이스 내 요약·단계 이동 허브.
- **첫 화면(처치 시작)**: **L1(음성·구급일지)로 바로 진입.** G1은 이후 뒤로가기·홈 진입 등으로 도달.
- **L1 ↔ G1**: **L1에서 G1으로 가는 뒤로가기(또는 동등 이탈)는 허용**하되, 이때 **[M3 출동 종료 경고]** 를 띄운다. **지금 출동을 완료한다**는 취지의 경고·확인 후 진행(확인 시 G1 등 목적지로 이동, 취소 시 L1 유지). G1 → L1 진입 시에는 동 경고 없음.
- **뒤로가기**: 시스템/앱 뒤로가기로 상위 화면 팝(단, L1→G1은 위 M3 규칙). M1/M2/M3/M4는 닫기·확인 후 이전 레이어로 복귀.
- **L4 진입 조건**: pre-KTAS·필수 증상이 최소 기준에 미달해도 **완전 차단하지 않음**. 진입 시 **경고**를 표시하고, 사용자가 **확인(진행)** 하면 L4로 진입. `screen-spec`에서 경고 문구·확인/취소 동작을 구체화.
- **pre-KTAS·L2 변경 → L4**: L2에서 pre-KTAS(및 증상 등 L4 추천에 영향 주는 항목)를 바꾸면, **L4 추천·카드 목록은 즉시 반영**(별도 “새로고침” 없이 동기). `screen-spec`에서 데이터 갱신 타이밍·로딩 표현만 구체화.

## 서비스 플로우

### 플로우 1. 현장 기록 → 평가 → 이송 조율 (정상)

```
[처치 시작]
  → [L1 음성·구급일지] (기록 자동 수집 가정)
  → (필요 시 G1 경유 없이) L2로 이동
  → [L2 증상·pre-KTAS] (자동 제안 확인·필요 시 수정·보완)
  → '병원 추천·이송' 등으로 이동
  → [L4 이송 병원 추천·조율]
      병원 A·B·C 등 **여러 카드에 동시에** [이송 신청] 가능 → 카드별 승인 상태 표시
      → **승인된 병원 중 한 곳** 카드에서만 [출발]
  → [M1 확인] (출발 최종 확인 — 정책에 따라 생략 가능, screen-spec에서 확정)
  → 확인
  → **출발 확정 처리**: 선택한 병원만 이송 진행, **나머지 병원에 대한 이송 신청은 자동 취소·철회**
  → [L4] 화면 유지, **이송 중** 상태로 전환 (G1 등으로 자동 이탈하지 않음)
  → (병원 도착) **도착 확인·완료 액션** 등
  → [M4 이송·케이스 완료 확인] — **완료할지** 질문
  → '완료' 확인 시 케이스 종료 후 **[G1 구급 홈]** 으로 이동
```

### 플로우 2. 네트워크·동기화 실패

```
[L4 이송 병원 추천·조율] 또는 [L1]/[L2] 등 전송 필요 시점
  → 전송 실패·오프라인 감지
  → [M2 동기화·오류] 또는 인라인 배너 + 재시도
  → 재시도 성공 시 해당 화면 상태 갱신
  → 사용자가 앱 종료·복귀 시 G1·L4 등에서 동기화·이송 상태 표시
```

### 플로우 3. 필수 정보 미충족 상태에서 L4 진입

```
[L2 증상·pre-KTAS]
  → 필수 미충족인 채 L4 진입 시도
  → 경고(다이얼로그/바텀시트 등): 리스크·보완 권고
  → '취소' → L2 유지 / '계속' 또는 동등 확인
  → [L4 이송 병원 추천·조율]
```

(L2 내부에서 필수 항목 저장 시도 시에는 기존처럼 인라인 오류로 L2 유지 가능 — `screen-spec`에서 구분.)

### 플로우 4. STT·NER 품질 저하

```
[L1 음성·구급일지]
  → 구간별 신뢰도 낮음 표시
  → 대원이 해당 구간 수동 편집·재녹음(정책 있을 시)
  → [L2]로 넘어가기 전 L1에서 최소 품질 기준 충족 유도
```

### 플로우 5. 이송 신청이 전부 실패(전원 거절·무응답 등)

```
[L4]
  → 신청한 병원이 **모두 실패**한 상태로 판단(기준은 screen-spec)
  → 대원은 **병원에 다시 이송 신청**하면 됨(동일 L4에서 다른 카드·재시도·목록 갱신 등)
  → 별도 “종료” 강제 없이 **재요청·재신청**으로 복구
```

### 플로우 6. L1에서 G1으로 뒤로가기

```
[L1 음성·구급일지]
  → G1으로 가는 뒤로가기(또는 동등 이탈)
  → [M3 출동 종료 경고] — 지금 출동을 완료하는지 안내
  → '취소' → L1 유지 / 확인 시 **출동 완료 처리** 후 [G1] (또는 정책상 목적 화면)
```

## 화면 간 이동 조건 요약

| 출발 | 도착 | 조건 |
|------|------|------|
| (처치 시작) | L1 | 세션/케이스 시작 시 기본 진입 |
| L1 | M3 | G1 등 상위로 이탈 시도(뒤로가기 등) |
| M3 | G1 | 사용자가 출동 완료 확인 |
| M3 | L1 | 취소 |
| L1 | L2 | 평가 단계 진입 액션 |
| L2 | L4 | 병원·이송 화면 진입 액션; 필수 미충족 시 **경고 확인 후** 진입. L2 변경분은 L4에 **즉시 반영** |
| L4 | M1 | 특정 카드 **출발** 탭(최종 확인이 있는 경우) |
| M1 | L4 | 확인 시 출발 확정·**타 병원 신청 자동 취소** 후 **L4 이송 중** / 취소 시 L4 유지(출발 전 상태) |
| L4 | M4 | 병원 **도착** 후 완료 확인 진입(트리거는 screen-spec) |
| M4 | G1 | **‘완료’** 확인 시 케이스 종료 후 이동 |
| M4 | L4 | **취소·미완료** 시 이송 중 화면 유지(표현은 screen-spec) |
| 임의(전송 시점) | M2 | 네트워크 오류·오프라인 |

## 병원 카드(UI) 행위 정리 (L4)

- **이송 신청**: 카드 단위로 신청. **여러 병원에 동시 이송 신청 허용**. 전원 실패 시 **같은 화면에서 병원에 다시 신청**(재요청)으로 대응.
- **승인**: 병원별로 독립적으로 승인 상태를 카드에 표시(텍스트·배지 등).
- **출발**: **승인된 병원 중 1곳**만 선택해 이송 시작. **출발 확정 시** 나머지 병원에 걸린 이송 신청은 **자동 취소·철회** 처리. 처리 후 **L4를 벗어나지 않고 이송 중 상태**로 표시.
- **도착·완료**: 병원 도착 후 **[M4]** 로 **케이스 완료 여부**를 묻는다. **완료** 선택 시 **[G1]** 으로 이동한다.
