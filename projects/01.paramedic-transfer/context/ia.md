## IA 구조

```
- [G1] 구급 일지  ← 허브
  - [L1] pre-KTAS / P-class·증상 (병렬·비동기)
  - [L2] 이송 병원 추천·검색·신청
    - [L3] 이송 승인·시작
      - [M1] 이송 시작 확인
        - [L4] 이송 시작 완료 (전용)
```

(L1과 L2는 **G1의 형제 진입** — L2는 L1 완료를 필수로 하지 않음.)

## 네비게이션 패턴

- **타입**: **허브(G1) + 스택** — G1에서 L1·L2로 갈라짐. L2→L3→M1→**L4(완료)** 단계형. (하단 **탭 바는 없음**; G1만 **하단 고정 액션 2개**로 평가·이송 진입)
- **G1 하단 고정**: `평가·증상(L1)` · `병원 추천·이송(L2)` — **항상 화면에 보이게**(리스트 스크롤과 분리; safe-area·키보드 대응은 스펙에서 세부화)
- **뒤로가기**: 상단 **스택 팝**. L1 비동기 처리 중에도 **G1으로 돌아가기** 허용(세션·작업 유지).
- **진입 불가 조건**:
  - L1: **일지 0건이어도 진입 허용** — G1은 **빈 상태 안내만**; L1에서 일지 미연계 안내( `L1` screen-spec )
  - L2: **G1에서 언제든 진입 가능**. **pre-KTAS 미확정·미완료여도 일괄 신청 허용**. **주증상·P-class(멀티 서브클래스)** 맥락만으로도 추천 요청 가능 — 상단 **안내·L1 유도**만 하고 **신청 버튼은 잠그지 않음**(선택 정책)
  - L3: **일괄 신청 완료 전**에는 L2 중심. **승인 0곳**이면 출발 불가 — 갱신·재신청 유도
- **오프라인·불안정 네트워크**: 화면별 **상태 배너·재시도**로 표현(별도 전역 화면 ID는 두지 않고 각 Global/Local 상단 또는 토스트 수준으로 설계 가능)

## 서비스 플로우

### 플로우 1-A. 일지 → L1(pre-KTAS ∥ P-class) → 추천·신청 → 승인 후 이송

```
[G1 구급 일지]  [정보: **본문** STT 소형·**일지 타임라인·수동 수정** 중심; **하단 고정** 평가·이송 **항상 노출**]
  → ‘평가·증상’ (하단 고정 CTA)
  → [L1]  [정보: 주·부증상, **증상 요약**; **pre-KTAS** 비동기 단계·등급·**근거 요약**·**필수 보완 필드**; **P-class** 서브클래스 멀티( **delivery** 기본 ☑, **Class III** 추천 연동)]
  → (처리 중) ← 뒤로 → [G1] 복귀 가능  [정보: 백그라운드 작업 표시(배지 등)]
  → 완료 후 사용자 보완·확정
  → L1에서 또는 G1에서 ‘병원 추천·이송’
  → [L2] 추천 탭  [정보: 추천 카드, 복수 선택]
  → 일괄 이송 신청 → [L3] → … (플로우 1-C와 합류)
```

### 플로우 1-B. 일지 → 병원 화면 직행(추천·검색) → 신청 → 승인 후 이송

```
[G1 구급 일지]  [정보: 동일 — 하단 고정에서 ‘병원 추천·이송’]
  → ‘병원 추천·이송’ (**L1·pre-KTAS 확정 생략 가능**)
  → [L2]  [정보: **pre-KTAS 미확정 안내**, 주증상·**P-class** 맥락; 추천 목록·**적합 요약·추천 사유·거리/이동**(제공 시)]
  → (선택) ‘검색’으로 전환  [정보: 검색어·필터, **검색 결과 병원**·식별·요약 — **추천과 동일 선택 풀**]
  → 추천·검색 결과에서 **동일 규칙**으로 복수 선택
  → 일괄 이송 신청
  → [L3] … (플로우 1-C)
```

### 플로우 1-C. 일괄 신청 이후 공통 (승인·출발)

```
[L2]
  → 병원 **여러 곳** 선택 후 일괄 신청  [정보: 선택 N·**신청 진행**·성공 시 **신청 시각·병원 ID 목록**; 실패 시 **오류 메시지**]
  → 신청 성공
  → [L3 이송 승인·시작]  [정보: 병원별 승인·거절·대기, 갱신 시각, **거절 사유**]
  → **승인된** 병원 1곳 선택 → ‘이송 시작’
  → [M1]  [정보: 병원명, 주증상·pre-KTAS·**P-class** 요약(가용 시)]
  → 확인
  → [L4 이송 시작 완료]  [정보: 최종 병원, 시각, 요약, G1 등 CTA]
```

### 플로우 2. STT·NER 오류 / 일지 항목 실패

```
[G1 구급 일지]
  → 특정 일지 항목 오류  [정보: 오류 사유, 재시도]
  → 재시도 또는 **수동 수정·보정** (항상 가능)
  → 일지 유지
```

### 플로우 3. L1 필수 미입력(L1 내부 검증)

```
[L1]
  → 필수 미충족 상태로 확정·저장 시도
  → 인라인 오류 (화면 유지)  [정보: 누락 필드명]
  → (참고) **pre-KTAS “확정”**은 L1 내부 규칙만 적용 — **L2 일괄 신청은 확정 없이 허용**(문서 정책)
```

### 플로우 4. 이송 신청 네트워크 실패

```
[L2]
  → ‘일괄 이송 신청’
  → 네트워크 오류  [정보: 실패 토스트/배너, 재시도]
  → [L2] 유지, 재신청
```

### 플로우 5. 승인 없음·전원 거절 등 재조정

```
[L3 이송 승인·시작]
  → 전원 거절 또는 유효 승인 없음  [정보: 병원별 거절·만료 상태, **거절 사유**]
  → ‘다른 병원 신청’ 또는 뒤로가기로 [L2] 복귀  [정보: 기존 선택·신청 이력(정책에 따라 초기화 또는 유지)]
  → [L2]에서 재선택·재신청
  → [L3]
```

## 대표 화면 와이어프레임

### G1 — 구급 일지

```
┌──────────────────────────────┐
│ 헤더: 구급 일지   [동기화]   │
├──────────────────────────────┤
│ [이송 중 · 병원명 …] (플로팅)│ ← L4 복귀 후, 탭 시 L4
├──────────────────────────────┤
│ [오프라인·재시도 배너]       │
├──────────────────────────────┤  ↑ 스크롤 영역 (일지 작업 중심)
│ ┌ STT·인식: 소형 한 줄 … ┐   │  ← 작게 (보조)
│ └ (파형/상태 아이콘 축소) ┘   │
├──────────────────────────────┤
│ ┌──────────────────────────┐ │
│ │ 시각·원문/추출·태그·상태 [편집]│ │  ← 수동 수정·표시 중심
│ └──────────────────────────┘ │
│            × N  (일지 메인)   │
├──────────────────────────────┤  ↓
│ ═════ 하단 고정 (항상 노출) ═│
│ [ 평가·증상(L1) ] [ 병원 추천·이송 ]│
└──────────────────────────────┘
```

**정보 밀도 점검**: **시각적 우선순위** — 본문은 **STT 소형 + 일지 카드(수동 수정)** 가 메인. 타임라인 **카드당** 시각·원문/추출·태그·상태·편집 **5필드 이상**. **평가·이송**은 **하단 고정 2버튼**으로 **항상 가시** — 스크롤로 숨기지 않음. **MVP NER**는 **환자증상·환자평가·응급처치** 키 우선.

### L1 — pre-KTAS / P-class·증상 (병렬·비동기)

```
┌──────────────────────────────┐
│ ← 평가·증상                  │
├──────────────────────────────┤
│ 주증상 [선택] → P-class 제안  │
│ 부증상                       │
│ [증상 요약] (비동기·수동 보완)│
├──────────────┬───────────────┤
│ pre-KTAS     │ P-class       │
│ [비동기단계] │ [비동기단계]  │
│ 등급·근거요약│ ▼ Class I    │
│ [필수보완필드]│  ☑☐☐☐ 서브  │
│              │ ▼ Class II (delivery 기본☑)│
│              │ ▼ Class III (추천 반영)│
│              │ … (멀티선택)  │
├──────────────┴───────────────┤
│ [ 병원 추천·이송 ]           │
└──────────────────────────────┘
```

**정보 밀도 점검**: 상단 **주·부증상 + 증상 요약**으로 카드 밖에도 **3필드 이상** 확보. pre-KTAS 블록에 **단계·등급·근거·필수 보완** **4요소** — 빈 영역 **50% 미만** 유지하려면 **스크롤·2열(세로 스택)** 로 밀도 분산. P-class는 **서브클래스 다수** → **Class별 아코디언·스크롤** 필수. 처리 중 **뒤로 → G1·L2** 가능.

### L2 — 이송 병원 추천·검색·신청

```
┌──────────────────────────────┐
│ ← 이송 병원                  │
├──────────────────────────────┤
│ [pre-KTAS 미확정 시 안내 배너]│
│ [미확정이어도 신청 가능·L1 유도]│
│ [이송 신청 대기 중 · 응답 보기→L3]│ ← 신청 제출 후
├──────────────────────────────┤
│ [ 추천 ] [ 검색 ]            │
├──────────────────────────────┤
│ 검색: [___________] [필터?]  │  ← 검색 탭 시
│ (또는 추천 리스트)            │
│ ┌ ☑ 병원명 (굵게)            │ │
│ │ 식별·위치·전문 요약 1~2줄  │ │  ← 너무 얇지 않게
│ │ 적합요약·추천사유          │ │
│ │ 거리·이동(제공 시)         │ │
│ └ …                      ┘   │
│            × N               │
├──────────────────────────────┤
│ 선택 N곳·신청 진행/오류 표시 │
│ [ 일괄 이송 신청 ]           │
└──────────────────────────────┘
```

**정보 밀도 점검**: **사용자 확정** — 카드를 **한 줄 병원명만** 두지 않고, **명칭 + 식별·위치·전문 등 요약 1~2줄 + 적합/사유 + 거리**(제공 시)로 **적정 밀도** 유지. **최소 5블록** 수준(체크·명칭·요약줄·적합·거리/사유)을 기본 가정. 상단 **배너 2줄**. **탭 2개**, **선택 풀 단일**. **최소 1곳 미선택** 시 CTA 비활성.

### L3 — 이송 승인·시작

```
┌──────────────────────────────┐
│ ← 이송 조율                  │
├──────────────────────────────┤
│ [갱신 시각·당겨서 새로고침 힌트]│
├──────────────────────────────┤
│ ┌──────────────────────────┐ │
│ │ 병원명                   │ │
│ │ 승인상태(대기|승인|거절) │ │
│ │ [거절 시 사유 텍스트]    │ │
│ │ 상태 시각                │ │
│ └──────────────────────────┘ │
│            × N               │
├──────────────────────────────┤
│ 승인 병원만 선택(라디오 등)  │
│ [ 이송 시작 ] (승인 0곳 시 비활성)│
└──────────────────────────────┘
```

**정보 밀도 점검**: 카드당 **병원명·승인상태·상태시각** **3필드** + 거절 시 **사유**로 **4필드** — 거절 없는 행은 **병원 식별 보조**(예: 신청 순번)로 **3미만 방지** 검토. 상단 **갱신 시각·당겨서 새로고침**으로 **빈 영역 ~15%** 수준 유지 목표.

### L4 — 이송 시작 완료 (전용)

```
┌──────────────────────────────┐
│ 이송이 시작되었습니다         │
├──────────────────────────────┤
│ 최종 병원명                  │
│ 확정 시각                    │
│ (주증상·KTAS·P-class 요약)   │
├──────────────────────────────┤
│ [ 구급 일지로 돌아가기 ]     │
└──────────────────────────────┘
```

**정보 밀도 점검**: **완료 전용**이라 정보는 **4블록**(제목·병원·시각·요약+CTA)으로 **빈 화면 50% 초과 방지** — 필요 시 **한 줄 안전 문구**(예: 현장 병원 연락 유지)는 스펙 단계에서 추가 검토. L3와 **역할 분리**(조율 vs 완료).
