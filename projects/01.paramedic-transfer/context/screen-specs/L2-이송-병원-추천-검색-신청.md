# L2 — 이송 병원 추천·검색·신청

## 개요
- 목적: 세션 맥락(일지·주증상·P-class·pre-KTAS 가용분)으로 **추천**과 **직접 검색**을 같은 선택 풀에서 다루고, **복수 선택 후 일괄 이송 신청**한다.
- 진입 경로: G1 하단 **「병원 추천·이송」**(pre-KTAS 확정 불필요). L1에서 **「병원 추천·이송」**. (선택) L1 보완 유도 링크.
- 이탈 경로: **일괄 신청 성공** → L3. **뒤로** → G1. **L3에서 뒤로** 복귀 시 **「이송 신청 대기 중」** 상태로 진입(아래). (선택) 평가 보완 → L1. 네트워크 실패 시 L2 유지(플로우 4).

## 검색 탭 — 필터 정의 (고정 스펙)

검색 결과 목록 상단 **필터 칩 + 「필터」시트** 조합. **추천 탭**은 세션 맥락으로 목록이 이미 좁혀지므로 **동일 필터를 선택 적용**할 수 있게 하되(일관성), 기본은 **검색 탭에서 편집**을 전제로 한다.

| 필터 ID | 라벨(예시) | 타입 | 옵션 / 동작 | 비고 |
|---------|------------|------|-------------|------|
| `sort` | 정렬 | 단일 선택 | **거리 가까운 순** · **적합도 높은 순**(서버 스코어) · **병원명 가나다** | 목록 재정렬 |
| `radius` | 거리 | 단일 선택 | **전체** · **5km** · **10km** · **20km** · **30km 이상은 직접입력**(숫자 km) | 현장·구급대 위치 기준(제공 시) |
| `specialty` | 전문/수용 | 다중 선택 | **응급의학** · **심장·혈관(PCI 등)** · **뇌·신경** · **외상·중증외상** · **소아** · **산부인과(분만)** · **화상** · **신장투석(CRRT 등)** · **중환자(ICU)** · **중재방사선(IR)** · **신생아(NICU)** | P-class·pre-KTAS와 **교차 강조**만 하고 자동 잠금은 하지 않음 |
| `accepting` | 수용 표시 | 단일 선택 | **전체** · **수용 가능만**(데이터 있을 때) · **미확인 포함** | 데이터 없으면 옵션 숨김 또는 비활성 |
| `hours` | 운영 | 단일 선택 | **전체** · **24시간 응급** | |
| `reset` | 초기화 | 액션 | 모든 필터·정렬 기본값 | 시트 하단 고정 |

- **추천 탭**: 상단에 **현재 필터 요약 칩**(적용 중일 때만) + 탭 시 시트 열기.
- **검색 탭**: 검색어 행 아래 **필터 칩 행** + **「필터」** 로 전체 시트.

## 레이아웃 구조
- 네비게이션: 상단 **뒤로** + 제목(이송 병원 등)
- 헤더: 배너 영역 — **pre-KTAS 미확정 안내**, **미확정이어도 신청 가능·L1 유도**(잠금 없음, ia). **신청 제출 후·응답 대기 중**이면 추가로 **「이송 신청 대기 중」** 한 줄(또는 배지) + **「병원 응답 보기」** → **L3** 보조 CTA.
- 주요 영역: **탭** — 「추천」 / 「검색」. 목록은 **체크 가능한 병원 카드**(적정 밀도)
- 하단 고정 영역: 있음 — **선택 N곳**, **신청 진행/오류** 표시, **「일괄 이송 신청」** CTA. **신청 대기 중** 세션에서는 하단 보조에 **「신청 대기 중 · 응답 보기」** 텍스트 버튼(또는 CTA 위 라벨)으로 L3 진입 유지

|--------|--------|------|------|
| 탭 전환 | 추천↔검색 | 동일 선택 풀 유지 | 선택 상태 보존 |
| 검색 실행 | 검색 버튼·엔터 | 결과 목록 갱신 | 검색어 입력 시 |

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 추천 목록 로드 완료 | 탭·카드·선택 수 표시 |
| 신청 대기 중 | L3에서 뒤로 또는 신청 직후 재진입 | **「이송 신청 대기 중」** 표기 + **응답 보기→L3**; 재선택·추가 신청 정책은 세션 규칙에 따름 |
| 로딩 | 추천/검색 요청 중 | 리스트 스켈레톤 |
| 빈 상태 | 결과 0건 | 빈 메시지 + 검색·필터 안내 |
| 오류 | 요청 실패 | 배너 + 재시도 |
| 신청 중 | 일괄 신청 진행 | CTA 로딩·중복 탭 방지 |
| 오프라인 | 연결 없음 | 배너 + 신청 제한 또는 큐 정책 |

## 엣지케이스
- pre-KTAS 없이 진입: 상단 안내만, 신청 버튼은 잠그지 않음(ia).
- 추천 탭과 검색 탭에 동일 병원 표시: 체크 상태 동기화(동일 선택 풀).
- 일괄 신청 중 앱 백그라운드: 재진입 시 진행 상태 복구 또는 타임아웃 안내.
- 필터 조합 결과 0건: 빈 상태 + 필터 초기화 제안.

## 패턴 후보
- **병원 선택 카드(체크 + 밀도 있는 본문)**: L3 목록과 **병원명·식별** 표현 통일 검토.
- **하단 선택 요약 + 단일 강한 CTA**: L3 이전 단계 공통 패턴.
