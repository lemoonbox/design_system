# L1 — pre-KTAS / P-class·증상 (병렬·비동기)

## 개요
- 목적: 주·부증상과 증상 요약을 다루고, **pre-KTAS**와 **P-class**를 **병렬·비동기**로 입력·확인하며, 필요 시 병원 단계로 넘어간다.
- 진입 경로: G1 하단 **「평가·증상」** → L1. 세션 일지 컨텍스트 전달(ia). **G1 일지 0건이어도 진입 허용** — G1은 빈 상태 안내만 하며 L1으로 막지 않음.
- 이탈 경로: **뒤로** → G1(비동기 작업 유지). **「병원 추천·이송」** → L2(pre-KTAS 확정 불필요). L1 내부 확정·저장 실패 시 화면 유지(플로우 3).

## 레이아웃 구조
- 네비게이션: 상단 **뒤로** + 제목(평가·증상 등)
- 헤더: 뒤로·타이틀 일체 또는 그 아래 컨텐츠 헤더
- 주요 영역: 혼합 — 상단 **주증상·부증상·증상 요약**; 하단부 **pre-KTAS 블록**과 **P-class 블록** 병렬(와이어: 세로 스택 또는 2열 스크롤)
- 하단 고정 영역: 있음 — **「병원 추천·이송」**(ia 와이어) + (필요 시) pre-KTAS/P-class **저장·확정** CTA는 블록 내 또는 하단 정책으로 배치

## 엣지케이스
- 처리 중 G1·L2로 이동: 허용; G1에서 배지 등 백그라운드 작업 표시(ia 플로우 1-A).
- pre-KTAS만 오류·P-class만 완료: 블록 독립 UI로 각각 재시도.
- Class III만 선택: 추천 반영 안내 표시; L2에서 맥락 전달(service-plan).
- delivery 기본 체크: 최초 제안 시 ON, 사용자 해제 가능.

## 패턴 후보
- **병렬 비동기 2컬럼(세로 스택) 카드**: pre-KTAS vs P-class — 다른 화면 재사용은 제한적이나 “상태 배지 + 근거 요약”은 L3·M1 요약과 문구 톤 통일.
- **Class 아코디언 + 멀티 체크**: P-class 마스터 표(service-plan)와 1:1 대응.
