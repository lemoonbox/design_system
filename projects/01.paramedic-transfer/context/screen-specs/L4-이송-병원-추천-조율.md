# L4 — 이송 병원 추천·조율

## 개요
- 목적: 추천 병원을 검토·다중 이송 신청·승인 상태 확인 후 한 곳만 출발 확정하고, 이송 중·도착 후 케이스 완료까지 같은 화면 흐름을 유지한다.
- 진입 경로: (ia.md) G1·L2에서 진입; L2 변경분은 **즉시** 목록에 반영.
- 이탈 경로: (ia.md) **M1** 후 출발 확정 시에도 **L4 유지(이송 중)**; **M4** 후 완료 시 **G1**; M4 취소 시 L4 유지; 임의 시점 **M2**(동기화·오류).

## 레이아웃 구조
- 네비게이션: 상단 뒤로·제목 “이송·병원”.
- 헤더: 있음 — 현재 모드 배지: **추천·신청** / **이송 중**; 동기화·오프라인 표시.
- 주요 영역: **병원 카드 세로 목록**(스크롤).
- 하단 고정 영역: 이송 중일 때 **“병원 도착”**(또는 동등) 등 M4 진입 트리거 CTA 권장.

## 엣지케이스
- **이송 신청 결과**: L2→L4 및 카드 상태는 **성공 시나리오**를 기본으로 둔다. **“전원 실패” 등 집계 판단 기준은 본 스펙에서 정의하지 않음**. 거절·무응답 카드는 **재신청** UI로 대응.
- **플로우 2**: 신청·취소·출발 확정 각각 실패 시 재시도; 앱 복귀 시 카드 상태와 G1 요약 정합.
- L2 필수 미충족으로 진입한 경우: L4 상단에 **경고 칩/배너** 유지(보완하려면 L2로 이동 링크).
- **M1**: **항상 표시**(출발 확정 전 최종 확인). 생략 없음.
