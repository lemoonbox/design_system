# L1 — 음성·구급일지

## 개요
- 목적: 음성 기반 STT·NER로 쌓인 구급 일지를 확인·보완하고, 평가 단계(L2)로 넘어갈 수 있게 한다.
- 진입 경로: (ia.md) 처치 시작·세션 시작 시 **기본 진입**; G1에서 “음성·구급일지” 진입.
- 이탈 경로: (ia.md) **G1 등 상위 이탈 시도** → **M3**; M3 확인 후 G1. **L2**로 평가 단계 진입 액션.

## 레이아웃 구조
- 네비게이션: 상단 뒤로(→ G1 시 M3) · 제목 “구급 일지” 등.
- 헤더: 있음 — 녹음/수집 상태(진행 중·일시정지 등 정책 반영), 동기화 표시(필요 시).
- 주요 영역: **시간순 발화 구간 목록**(각 구간: 원문 텍스트·메타).
- 하단 고정 영역: 있음 — **“증상·pre-KTAS로”** 다음 단계 CTA; 마이크·수집 제어는 정책에 따름. **구간 재녹음 UI·재녹음 액션은 제공하지 않음**(텍스트 편집만).

## 액션
| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 구간 편집 | 구간 탭·편집 아이콘 | 인라인/전체 화면 편집 | 항상 |
| 구간 저장 | 저장·완료 | 로컬 반영·전송 큐 | 오프라인 시 큐잉 |
| 상위(뒤로)로 이동 시도 | 시스템/앱 뒤로 | **M3** 표시 | G1 등 상위 이탈일 때 |
| M3 취소 | 모달 취소 | L1 유지 | — |
| M3 확인(출동 완료) | 모달 확인 | 출동 완료 처리 후 **G1** | 정책상 문구는 M3 스펙 따름 |
| 증상·pre-KTAS로 | 하단 CTA | **L2** 이동 | **항상 허용**(별도 최소 품질·구간 개수 게이트 없음) |
| 동기화 재시도 | 배너·메뉴 | **M2** 또는 백그라운드 재시도 | 전송 실패 시(플로우 2) |

## 상태 정의
| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 구간 목록 로드 완료 | 목록 렌더 |
| 로딩 | 초기·동기 후 갱신 중 | 스켈레톤 |
| 빈 상태 | 아직 구간 없음 | “말로 기록이 시작됩니다” 등 안내 |
| 오류 | 로드·저장 실패 | 메시지 + 재시도 |
| 수집 중 | STT 수신 중 | 상단 인디케이터·구간 실시간 추가 |
| 저신뢰도 안내(선택) | 신뢰도 낮은 구간 존재 | 배지·문구로 **편집 권장**만 표시; **L2 진입은 차단하지 않음** |

## 엣지케이스
- **STT·NER 품질**: 저신뢰 구간은 **텍스트 편집**으로 보완만 안내. **재녹음 없음**.
- **플로우 2**: 저장·전송 실패 시 M2 또는 배너; 앱 복귀 시 G1·L4 등과 동일하게 동기 상태 표시 가능.
- **L2 진입**: 최소 품질·최소 구간 수 등 **게이트 없음**(본 제품 가정).

## 패턴 후보
- 타임라인 구간 행: 다른 로그형 화면과 공통 가능성 낮음(도메인 특화).
- 상단 동기화 배너: L2·L4와 공통.
