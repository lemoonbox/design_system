# L3 — 이송 승인·시작

## 개요
- 목적: 일괄 신청한 병원별 **승인·거절·대기** 상태를 보고, **승인된 병원 중 1곳**만 골라 **이송 시작**으로 이어간다.
- 진입 경로: L2에서 **일괄 이송 신청 성공** 후. **「병원 응답 보기」** 등으로 L2에서 재진입.
- 이탈 경로: **승인 병원 1곳 선택 + 이송 시작** → M1. M1 **취소** → L3 복귀. **전원 거절·승인 0** 등 → **다른 병원 신청** 또는 뒤로 **L2**(플로우 5). **뒤로 → L2** 시 L2에 **신청 대기 중** 상태 반영(L2 스펙).

## 레이아웃 구조
- 네비게이션: 상단 **뒤로** + 제목(이송 조율 등). **뒤로 버튼 영역**에 **보조 라벨** 표기 — **신청 제출 후·병원 응답 대기 중**이면 **「이송 신청 대기 중」**(또는 짧게 **「신청 대기 중」**)을 **버튼 아래 한 줄** 또는 **뒤로 칩**으로 표시. **신청 전**에는 해당 문구 없음(L3는 신청 후에만 진입하므로 L3 내에서는 기본적으로 **대기 맥락** 노출; **전 병원 응답 완료**(대기 0건) 후에는 보조 라벨을 **「응답 확인됨」** 등으로 전환하거나 생략).
- 헤더: **갱신 시각**, **당겨서 새로고침** 힌트(ia)
- 주요 영역: 병원별 **카드 목록**(상태·시각·거절 사유) + **승인 병원 단일 선택(라디오 등)**
- 하단 고정 영역: 있음 — **「이송 시작」** (승인 0곳 시 비활성)

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 당겨서 새로고침 | 제스처 | 병원별 상태 재조회 | 네트워크 가능 시 |
| 승인 병원 선택 | 라디오·행 탭 | 선택 ID 설정 | 상태=승인 인 행만 |

| 뒤로 | 상단 | L2 | **항상** 스택 팝; L2에서 **「이송 신청 대기 중」** UI 표시(신청은 이미 제출된 상태). 보조 라벨은 위 **데이터 필드** 참조 |

## 엣지케이스
- 일부만 승인: 라디오는 승인 행만; 거절 행에 사유 표시.
- 동시에 여러 병원 승인: 사용자는 **1곳만** 선택 가능 — 선택 변경 허용.
- 만료 상태: 출발 후보에서 제외, 설명 카피.
- 재진입 L3: 직전 선택 유지 여부 정책 결정.

## 패턴 후보
- **상태 배지 + 시각 + 조건부 사유 블록**: 이후 운영 화면 확장 시 재사용 가능.
- **단일 선택 + 하단 확정 CTA**: M1과 연속 동선에서 톤 맞춤.
