# G1 — 구급 일지

## 개요
- 목적: 현장에서 음성·STT·NER로 구급 일지를 누적·검토·수정하고, 평가(L1) 또는 병원 추천·이송(L2)으로 이어간다.
- 진입 경로: 앱 세션 허브(처치 시작과 함께 기록이 시작된다는 전제). 타 화면에서 `구급 일지로` 복귀 시(L4 등).
- 이탈 경로: 하단 고정 **「평가·증상」** → L1; 하단 고정 **「병원 추천·이송」** → L2( pre-KTAS 확정 불필요, ia.md·service-plan 동일). L4에서 **「구급 일지로」** 복귀 시 **이송 중 플로팅** 노출(아래 레이아웃).

## 레이아웃 구조
- 네비게이션: 상단 앱 바(뒤로 없음 또는 정책에 따름) + 헤더 영역 내 **동기화** 등 액션
- 헤더: 있음 — 화면명(구급 일지), 동기화/상태 진입(아이콘 또는 텍스트)
- **이송 중 플로팅**(조건부): 헤더·배너 **아래**, 스크롤 콘텐츠 **위**에 **고정(플로팅) 바** 1줄 — L4 복귀 직후 **이송 진행 맥락** 표시, **탭 시 L4**로 이동(동일 세션 요약 재진입). 일지 스크롤과 겹치지 않게 z-index·safe-area 처리.
- 주요 영역: **스크롤** 타임라인(일지 카드 목록) + 상단 **STT·인식 소형(보조 한 줄)** (플로팅이 있으면 그 아래부터 스크롤 시작)
- 하단 고정 영역: 있음 — **「평가·증상(L1)」**, **「병원 추천·이송(L2)」** (스크롤과 분리, 항상 노출; safe-area·키보드 대응은 구현 세부)

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 음성 입력·STT 시작/유지 | 마이크·정책 UI | 일지 항목 후보 생성·갱신 | 권한·네트워크 정책 |
| 일지 항목 수동 수정 | 카드 편집 | 값 저장(로컬/동기) | 항상 허용(service-plan) |
| NER·매핑 재시도 | 재시도 탭 | 동일 항목 파이프라인 재실행 | 오류 상태일 때 |
| 동기화 | 헤더·배너 | 큐 플러시 시도 | 오프라인 복귀 등 |
| 평가·증상 | 하단 고정 버튼 | L1으로 이동 | **일지 0건이어도 진입 허용** — G1은 **빈 상태 안내만** 하고 CTA는 **비활성화하지 않음** |
| 병원 추천·이송 | 하단 고정 버튼 | L2로 이동 | pre-KTAS 확정 불필요(ia) |
| 이송 중 플로팅 탭 | 플로팅 바 전체 | **L4**(이송 시작 완료 요약)로 이동 | 플로팅이 **표시 중**일 때만 |
| 이송 플로팅 닫기(선택) | 스와이프·X | 바 숨김 | 정책: **다음 케이스**·명시적 종료 시에도 숨김; 닫아도 L4는 스택/딥링크로 재진입 가능하면 UX 유지 |

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 세션·일지 로드 완료 | 타임라인·STT 소형·하단 CTA 정상 |
| 로딩 | 초기·동기 후 갱신 중 | 스켈레톤 또는 상단 진행 |
| 빈 상태 | 일지 항목 0건 | 안내 문구 + 음성 시작 유도; **평가·증상 CTA는 그대로 활성**(L1은 빈 맥락에서도 진입) |
| 이송 플로팅 표시 | L4→G1 복귀 직후·동일 케이스 이송 진행 중 | 플로팅 바 노출 |
| 이송 플로팅 숨김 | 다음 케이스·세션 리셋·사용자 닫기(정책) | 바 제거 |
| 오류 | 네트워크·STT/NER 실패 | 항목별 메시지 + 재시도; 전역은 배너 |
| 오프라인 | 네트워크 단절 | 배너 + 로컬 큐 표시 |
| 편집 중 | 필드 포커스 | 저장·취소 또는 자동 저장 정책 |

## 엣지케이스
- STT/NER 특정 항목만 실패: 해당 카드에 오류·재시도, 나머지 타임라인은 유지(플로우 2).
- 오프라인에서 입력·수정: 큐잉 후 동기화 실패 시 재시도 UI.
- 일지 전무 상태에서 L1: **진입 제한 없음** — G1 빈 상태 카피만 강화(L1 내부에서 주증상·일지 유도).
- 플로팅과 STT 소형 영역이 겹침: 플로팅 우선, STT는 그 아래로 밀기.
- 키보드가 하단 CTA 가림: safe-area·키보드 인셋 처리.

## 패턴 후보
- **일지 타임라인 카드**: 시각·원문/추출·태그·상태·편집을 한 카드에 묶는 패턴 — L1·L2와 직접 공유하진 않으나 허브 헤더·배너는 타 화면과 톤 통일.
- **하단 이중 고정 CTA**: G1 전용; 다른 Local 화면과 패턴 분리.
- **허브 플로팅 → 완료 화면 재진입**: L4 요약을 G1에서 한 탭으로 복귀시키는 패턴.
