# M1 — 확인 (출발 최종 확인)

## 개요
- 목적: 이송 출발 확정 전 선택 병원·자동 취소될 타 신청을 한 번 더 확인한다.
- 진입 경로: (ia.md) L4에서 **승인된 병원 카드**의 **출발** 탭 → **항상 M1 표시**(생략 없음).
- 이탈 경로: (ia.md) **확인** → 출발 확정·타 병원 신청 자동 취소·**L4 이송 중**; **취소** → **L4** 유지(출발 전).

## 레이아웃 구조
- 네비게이션: 모달(다이얼로그/바텀시트). 배경 딤.
- 헤더: 있음 — 제목 “이송 출발 확인” 등.
- 주요 영역: **선택 병원명**, **다른 병원 신청이 자동 취소됨** 안내 문구(불릿).
- 하단 고정 영역: 있음 — **취소** · **출발 확정**(주요 버튼).

## 데이터 필드
| 필드명 | 타입 | 필수 여부 | 표시 방식 | 비고 |
|--------|------|----------|----------|------|
| 출발 병원명 | text | Y | 강조 텍스트 | |
| 자동 취소 대상 병원 목록 | list(text) | N | 목록 | 신청 중이던 타 병원 |

## 액션
| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 취소 | 보조 버튼 | 모달 닫힘·L4 출발 전 | 항상 |
| 출발 확정 | 주요 버튼 | 전송 성공 시 처리 후 L4 이송 중 | 네트워크 실패 시 M2·재시도 |
| 바깥 탭·뒤로 | 시스템 | 취소와 동등 처리 | 플랫폼 정책에 맞춤 |

## 상태 정의
| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 모달 오픈 | 확인 문구 표시 |
| 전송 중 | 확정 탭 직후 | 버튼 로딩·중복 탭 방지 |
| 실패 | API 오류 | 오류 문구 + 재시도 또는 M2 |
