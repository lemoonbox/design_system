# M1 — 이송 시작 확인

## 개요
- 목적: **승인 병원 1곳**에 대해 이송 출발 직전 **최종 맥락(주증상·pre-KTAS·P-class)** 을 확인하고 확정 또는 취소한다.
- 진입 경로: L3에서 **승인 병원 1곳 선택** 후 **「이송 시작」** 탭.
- 이탈 경로: **확인(확정)** → L4. **취소** → L3 복귀(ia).

## 레이아웃 구조
- 네비게이션: 모달 오버레이(시스템 뒤로 = 취소 정책과 정합)
- 헤더: 모달 타이틀(이송 시작 확인 등)
- 주요 영역: 단일 콘텐츠 — **병원명**, **주증상·pre-KTAS·P-class 요약**(가용 시)
- 하단 고정 영역: 있음 — **취소**, **확인(이송 시작 확정)**

## 데이터 필드

| 필드명 | 타입 | 계층 | 필수 여부 | 표시 방식 | 비고 |
|--------|------|------|----------|----------|------|
| 확인(확정) | action | 1차 | Y | 주 버튼 | L4로 이행 |
| 취소 | action | 1차 | Y | 보조 버튼 | L3 복귀 |
| 병원명 | text | 1차 | Y | 모달 본문 강조 | 선택 병원 ID에 대응 |
| 주증상 요약 | text | 2차 | N | 본문 | 가용 시 |
| pre-KTAS 요약 | text | 2차 | N | 본문 | 미확정이면 “미확정” 등 표기 정책 |
| P-class 선택 요약 | text | 2차 | N | 칩·한 줄 | 멀티 선택 목록 압축 |
| 일지 맥락 한 줄(선택) | text | 2차 | N | 캡션 | 최종 확인 보조 |
| 세션·요청 ID(내부) | text | 3차 | N | 숨김·디버그 | 지원용 선택 |

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 취소 | 버튼·바깥 탭(정책) | L3, 선택 상태 유지 | — |
| 확인 | 버튼 | 이송 시작 요청 → 성공 시 L4 | 네트워크 성공 시; 실패 시 모달 내 오류 |
| 배경 탭 | 오버레이 | (정책) 취소와 동일 또는 무시 | UX 정책으로 통일 |

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 모달 오픈 | 요약·버튼 활성 |
| 로딩 | 확정 API 진행 | 확인 버튼 로딩·중복 방지 |
| 오류 | API 실패 | 인라인 메시지 + 재시도 |
| 오프라인 | 연결 없음 | 확정 불가 안내 |

## 엣지케이스
- pre-KTAS·P-class 일부만 가용: 있는 필드만 표시, 없는 항목은 라벨 생략 또는 “—”.
- 확정 직후 L4 자동 전환: 조율 화면(L3)에 머무르지 않음(ia).
- 이중 탭 방지: 요청 중 버튼 비활성.

## 패턴 후보
- **확인 모달(요약 + 이중 버튼)**: 삭제·중단 등 고위험 액션에 변형 복제 가능(본 앱 범위外 참고).
