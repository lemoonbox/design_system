# M2 — 동기화·오류

## 개요
- 목적: 오프라인·전송 실패 시 사용자에게 상태를 알리고 재시도·대기 선택을 제공한다.
- 진입 경로: (ia.md) L4·L1·L2 등 **전송 필요 시점**에서 실패·오프라인 감지 시; 또는 인라인 배너에서 “자세히” 진입.
- 이탈 경로: 닫기·재시도 성공 후 **호출 스택 복귀**(이전 화면); 재시도 실패 시 모달 유지 또는 갱신.

## 레이아웃 구조
- 네비게이션: 모달 또는 풀스크린 단순 오류(제품 일관 선택).
- 헤더: 있음 — “연결할 수 없습니다” / “동기화 대기 중” 등 상황별 제목.
- 주요 영역: **원인 요약**(사용자 친화 문구), **영향**(예: “일지는 기기에 저장됨”).
- 하단 고정 영역: 있음 — **닫기** · **다시 시도**(주요).

## 데이터 필드
| 필드명 | 타입 | 필수 여부 | 표시 방식 | 비고 |
|--------|------|----------|----------|------|
| 오류 유형 | select | Y | 내부 | offline / timeout / server_error |
| 사용자 메시지 | text | Y | 본문 | 기술 스택 노출 금지 |
| 대기 중 작업 수 | number | N | 텍스트 | “전송 대기 3건” |
| 마지막 시도 시각 | date | N | 메타 | |

## 액션
| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 다시 시도 | 주요 버튼 | 큐 재전송·성공 시 닫힘 | 네트워크 복구 시 |
| 닫기 | 보조 버튼 | 모달 닫기·백그라운드 큐 유지 | 항상 |
| 앱 복귀 후 자동 재시도 | 시스템 | (선택) 조용히 재시도 | 제품 정책 |

## 상태 정의
| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 모달 오픈 | 메시지 + 버튼 |
| 재시도 중 | 버튼 탭 | 로딩 |
| 부분 성공 | 일부만 전송 | 남은 건수 표시 |
| 완료 | 전부 성공 | 토스트 후 닫힘 |

## 엣지케이스
- **플로우 2**: 재시도 성공 후 해당 화면(L4 카드·L1 일지) 상태 갱신.
- 장시간 오프라인: G1·L4에서 배지로 누적 표시(본 모달과 병행).
- 중복 모달: 동일 오류 연속 시 하나로 합치거나 배너로 대체(UX 정책).

## 패턴 후보
- 네트워크 오류 모달: 범용 앱 패턴으로 타 기능과 공통.
