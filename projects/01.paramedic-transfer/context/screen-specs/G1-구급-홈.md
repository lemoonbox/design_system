# G1 — 구급 홈

## 개요
- 목적: 진행 중 케이스의 단계·요약을 한눈에 보고, **주 행동은 음성·구급일지(L1)** 로 이어지게 한다. L2·L4는 보조 진입만 둔다.
- 진입 경로: (ia.md) M3에서 출동 완료 확인 후; M4에서 케이스 완료 확인 후; L2·L4 등에서 홈(또는 동등 상위)으로 이동 시(해당 화면의 백 스택 규칙에 따름). **L1에서의 G1 진입은 M3를 통해서만** 가능.
- 이탈 경로: (ia.md) L1·L2·L4로 이동 가능. G1 → L1 시 **M3 없음**. **하단 단일 주요 CTA는 L1만**(L2·L4는 스텝퍼/요약 등 보조 진입).

## 레이아웃 구조
- 네비게이션: 상단 앱 바(제목·케이스 식별) 또는 동등 구조. 하단 탭은 사용하지 않음(단계형 스택 가정).
- 헤더: 있음 — 현재 케이스/출동 식별(시간·케이스 라벨 등), 필요 시 동기화 상태 아이콘(오프라인·대기).
- 주요 영역: **요약 카드 + 단계 진행(체크리스트/스텝퍼)** 혼합. L2·L4는 **스텝/카드 탭** 등 **보조** 진입으로만 제공(주요 버튼 3개 나열 금지).
- 하단 고정 영역: **있음** — **단일 주요 CTA**: **「음성·구급 일지」→ L1** (시각적 위계 최상).

| 필드명 | 타입 | 필수 여부 | 표시 방식 | 비고 |

| 단계: 음성·구급일지 | select | Y | 스텝·아이콘 | 미시작·진행중·완료 |

## 액션
| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 음성·구급일지로 이동(주요) | **하단 단일 CTA** | L1로 이동 | 항상(M3 없음) |
| 증상·pre-KTAS로 이동(보조) | 스텝퍼·요약 단계 탭 등 | L2로 이동 | 항상 |
| 이송 병원·조율로 이동(보조) | 스텝퍼·요약 단계 탭 등 | L4로 이동 | 항상(L2→L4 필수 미충족 경고는 L2 규칙 따름) |
| 동기화 재시도 | 동기화 배너·아이콘 탭 | M2 표시 또는 인라인 재시도 | 전송 실패·오프라인 감지 시(플로우 2) |

## 상태 정의
| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 케이스 로드 완료 | 단계·요약 정상 표시 |
| 로딩 | 케이스·요약 조회 중 | 스켈레톤 또는 스피너 |
| 빈 상태 | 진행 중 케이스 없음(정책 허용 시) | “진행 중 출동이 없습니다” + L1 진입 안내 |
| 오류 | 조회 실패 | 오류 메시지 + 재시도 |
| 오프라인·동기화 대기 | 네트워크 불안 | 배지/배너 + M2 또는 재시도 진입 |

## 엣지케이스
- **L1에서만** G1 이탈 시 M3 적용; G1 자체에서는 해당 모달 없음.
- 오프라인 시에도 단계 요약은 **로컬 캐시** 기준 표시, 동기화 실패는 M2·배너로 안내(플로우 2).
- L4에서 이송 중인 경우 G1의 이송 단계는 “이송 중”으로 표시하고, 세부는 L4에서 확인하도록 안내 카피 병행 가능.

## 패턴 후보
- 단계형 스텝퍼/체크리스트: L2·L4 상단에도 축약 재사용 가능.
- 케이스 헤더(시각·동기화): L1·L2·L4와 동일 톤 권장.
