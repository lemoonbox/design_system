# PAT-001 — 비동기·상태 카드

## 메타
- 등장 화면: G1 (구급 일지 타임라인 항목), L1 (pre-KTAS / P-class 블록 카드), L3 (병원 응답 카드 내 상태 영역과 톤 통일)
- 역할: 비동기 파이프라인(일지 처리·pre-KTAS·P-class·병원 응답 등)의 **현재 단계·시각·근거·오류**를 한 카드 블록에 묶어 현장에서 빠르게 스캔·대응할 수 있게 한다
- 반복 구조: 타임라인·블록 목록에서 동일 프레임이 상태 값만 바뀌며 반복된다

## 시각적 의도

> 이 패턴은 **왼쪽 얇은 상태 띠가 달린 흰 카드**처럼 보인다. 사용자의 시선은 먼저 좌측 세로 컬러 띠와 상태 칩으로 가고, 이어 타임스탬프·본문·재시도 행으로 흐른다.

- **전체 인상**: 의료 도구 톤을 유지한 구조적 카드(border 1px). 카드 전체를 상태색으로 칠하지 않고 **좌측 4px 바 + 칩**으로만 상태를 말한다 (visual-direction 색상 면적 예산 준수)
- **시각적 초점**: 상태 칩(label) + 좌측 상태 바 — 성공/경고/오류/진행(primary) 구분
- **시각적 무게**: 상태 영역 35% : 본문·메타 65%
- **디자인 핵심**: `surface` 흰 카드 위에 `border — gray.300` 으로 경계를 고정하고, 조건부 블록(근거 요약·오류 사유·재시도)은 본문 계층(body / caption)으로만 구분한다

---

## Visual Recipe

스크롤 목록 속에서 흰 직사각형 한 장이 연한 회색 배경과 1px 테두리로 떨어져 보이고, 왼쪽 가장자리에만 4px 짜리 세로 컬러 띠가 붙어 “이 카드의 건강 상태”를 즉시 알려 준다. 상단 한 줄에는 작은 상태 칩과 시계 아이콘·타임스탬프가 나란히 있어 비동기 작업의 최신 시각을 확인할 수 있다. 본문은 짧은 제목(있을 경우)과 회색 톤 보조 설명으로 이어지고, 오류·재시도가 필요할 때만 아이콘과 버튼이 한 줄 더 붙어 행동을 유도한다. 전체적으로 친근·활동적 톤은 색의 **소면적 사용**과 굵은 제목보다는 읽기 좋은 본문 위계로 표현한다.

## 레이아웃 구조

### 전체 프레임 `비동기-상태-카드`

| 속성 | 값 | 설계 의도 |
|------|-----|----------|
| 크기 | 390 × auto (최소 약 96px) | 목록 폭에 맞추고 내용에 따라 높이 확장 |
| 배경 | surface — white #FFFFFF (rgba(255,255,255,1)) | 페이지 background와 휘도 대비 |
| 배치 | 수평 스택 — 좌측 고정 바 + 우측 콘텐츠 컬럼 | 상태를 형태(세로 띠)로 먼저 인지 |
| 주축 정렬 | 시작 | 바는 상단부터 하단까지 동일 높이 |
| 교차축 정렬 | 늘리기(stretch) | 좌측 바가 카드 전체 높이를 덮음 |
| 내부 여백 | 우측 콘텐츠 영역만 상16 하16 좌12 우16 — spacing.xl 기준, 바와의 간격 12px | 터치·가독 균형 (visual-direction 중간 밀도) |
| 요소 간격 | 세로 스택 gap 12px — spacing.lg | 블록 단위 스캔 |
| 모서리 | 10px — radius.lg | 카드 단위 통일 |
| 깊이 | 없음 — shadow 미사용 | 테두리로만 구분 (shadow·stroke 동시 사용 금지) |
| 테두리 | 전체 stroke — border gray.300 #D5D7DA (rgba(213,215,218,1)) 1px | 구조적 경계 |

### 구조 다이어그램

```
┌──┬────────────────────────────────────────────┐  390×auto
│██│  [상태 칩]  🕐 2025-03-15 14:32            │  ██ = 좌측 4px 상태 바
│██│  (선택) 부제목 / 항목명                     │
│██│  본문·근거·요약                             │
│██│  (조건부) 오류 문구 + 재시도                │
└──┴────────────────────────────────────────────┘
```

| 모서리 | 좌측 상하만 radius.lg에 맞춤 clip | 카드 외곽선과 정렬 |

## 시각 품질 검증

### Surface Depth

| 요소 | fill | 부모/배경 fill | 대비 | 조치 |
|------|------|--------------|------|------|
| 카드 | surface #FFFFFF | background #FAFAFA | 충분 | border 1px 유지 |
| (L1 등) 인접 입력 | surface-raised #F5F5F5 | surface #FFFFFF | 충분 | design-system 입력면 계층 |

### Elevation

| 요소 | 유형 | shadow 토큰 | 근거 (visual-direction) |
|------|------|------------|----------------------|
| 카드 | 목록 카드 | 없음 | 구조적 1px border 우선, 레이어 겹침 아님 |

### 타이포 위계

| 수준 | 역할 | 스펙 (크기/행간/자간/굵기) | 적용 요소 |
|------|------|-------------------------|----------|
| 1 | subheading | 18px/28px/0px/600 | 블록 제목(선택) |
| 2 | body | 16px/24px/0px/400 | 본문·근거 |
| 3 | label | 14px/20px/0px/500 | 상태 칩 |
| 4 | caption | 12px/18px/0px/400 | 타임스탬프·오류 보조 |

### 강조

- focal point: 좌측 4px 바 + 상태 칩 — 색은 상태 전용(success/warning/error/primary)으로 역할 분리
- 상태 표현: **바 + 칩 문구 + (선택) 아이콘** 병행
- 근거: visual-direction.mdc > 색상이 허용되는 기능적 위치(카드 좌측 상태 바·상태 배지)

### 밀도

- 카드 최소 높이로 인한 터치: 행 전체가 탭 타겟이면 44px 이상 확보
- ❌ 칩·아이콘만 단독 타겟일 경우: 주변 패딩으로 44×44 이상 확보 권장
