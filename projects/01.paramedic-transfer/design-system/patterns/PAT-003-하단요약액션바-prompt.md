# PAT-003 — 하단 요약 액션 바

## 메타
- 등장 화면: L2 (선택 N곳 + 일괄 이송 신청), L3 (승인 병원 선택 요약 + 이송 시작)
- 역할: 목록에서 한 번에 내린 선택·신청 맥락을 **한 줄 요약**으로 고정 표시하고, 화면당 **주 CTA 1개**로 다음 단계를 밀어 준다
- 반복 구조: L2·L3 하단 safe-area 위 고정 바로 동일 레이아웃, 문구·버튼 라벨·활성 조건만 변경

## 시각적 의도

> 이 패턴은 **화면 하단에 붙은 흰 액션 바**처럼 보인다. 사용자의 시선은 요약 숫자·문구로 현재 선택 상태를 확인한 뒤, 곧바로 넓은 보라색 CTA로 이동한다.

- **전체 인상**: 콘텐츠와 분리된 고정 영역. 상단 1px border로 스크롤 영역과 단절
- **시각적 초점**: 주 CTA 버튼 — brand.600 fill, 화면당 primary 면적 예산의 중심
- **시각적 무게**: 요약 텍스트 25% : CTA 75% (너비 기준 CTA 우선)
- **디자인 핵심**: 보조 링크·텍스트 버튼은 **accent blue.600** 또는 body + 밑줄로만 처리해 CTA와 역할 충돌을 피한다

---

## Visual Recipe

스크롤되는 병원 목록 아래에는 항상 같은 높이의 흰 바가 붙어 있고, 위쪽 가장자리만 얇은 회색 선으로 본문과 나뉜다. 왼쪽 또는 상단 한 줄에는 "N곳 선택" 같은 짧은 요약이 작은 글씨로 상태를 알려 주고, 그 아래(또는 같은 행의 우측)에는 가로로 긴 보라색 버튼이 자리를 차지해 손가락이 닿기 쉬운 영역을 확보한다. 신청 대기 중일 때는 같은 바 안에 텍스트 링크만 한 줄 더 얹어 L3으로 다시 들어가는 동선을 잃지 않게 한다. 전체적으로 장식 없이 **한 가지 강한 행동**만 남기는 현장용 레이아웃이다.

## 레이아웃 구조

### 전체 프레임 `하단-요약-액션-바`

| 속성 | 값 | 설계 의도 |
|------|-----|----------|
| 크기 | 390 × auto (최소 80px + safe-area) | 키보드·홈 인디케이터 대응 |
| 배경 | surface — white #FFFFFF (rgba(255,255,255,1)) | 본문과 동일 surface로 정리 |
| 배치 | 수직 스택 | 요약 → CTA → (선택) 보조 링크 |
| 주축 정렬 | 시작 | 좌우 padding spacing.xl |
| 교차축 정렬 | 늘리기(fill) | CTA full width |
| 내부 여백 | 상12 하12 좌16 우16 — 하단은 safe-area 추가 시 하단만 증가 | 터치·엄지 존 |
| 요소 간격 | 12px — spacing.lg | CTA와 요약 분리 |
| 모서리 | 없음 | 화면 하단 플러시 |
| 깊이 | 없음 | 바텀시트가 아니면 shadow 미사용 |
| 테두리 | 상단만 border — gray.300 #D5D7DA 1px | 고정 바 분리 |

### 구조 다이어그램

```
───────────────────────────────  border top
  선택 3곳 · 신청 진행 중…     ← summary (caption/body)
┌────────────────────────────┐
│     일괄 이송 신청           │  ← primary CTA
└────────────────────────────┘
  병원 응답 보기 →             ← (선택) accent 텍스트 링크
════════ safe area ══════════
```

## 시각 품질 검증

### Surface Depth

| 요소 | fill | 부모/배경 fill | 대비 | 조치 |
|------|------|--------------|------|------|
| 바 | white | 목록 영역(밝은 배경) | 충분 | 상단 border 필수 |
| 비활성 CTA | gray.400 | white | WCAG 대비 검증 | 라벨 가독성 확인 |

### Elevation

| 요소 | 유형 | shadow 토큰 | 근거 (visual-direction) |
|------|------|------------|----------------------|
| 하단 바 | 고정 액션 | 없음 | 레이어 겹침이 아니면 shadow 불필요 |

### 타이포 위계

| 수준 | 역할 | 스펙 | 적용 요소 |
|------|------|------|----------|
| 1 | button | 16px/24px/0px/600 | CTA |
| 2 | label | 14px/20px/0/500 | 선택 수 강조 |
| 3 | caption | 12px/18px/0px/400 | 보조 상태 |

### 강조

- focal point: 주 CTA — 화면당 primary fill **1곳**(visual-direction primary 면적 예산)
- 보조: accent 링크만 별색 — CTA와 목적 분리
- 근거: visual-direction.mdc > 강조 방식, 색상 분배 규칙

### 밀도

- CTA 높이 48px — 44px 터치 ✅
- 요약 행과 CTA 간 12px — 오동작 방지
