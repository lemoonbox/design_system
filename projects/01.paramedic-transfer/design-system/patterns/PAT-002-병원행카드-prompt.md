# PAT-002 — 병원 행 카드

## 메타
- 등장 화면: L2 (추천·검색 목록 — 체크 선택), L3 (응답 목록 — 라디오 단일 선택 + 승인 상태)
- 역할: 병원 1곳을 **식별·비교**할 수 있게 요약 정보를 제시하고, L2에서는 다중 선택, L3에서는 상태·사유와 함께 출발 병원을 고르는 데 쓴다
- 반복 구조: 두 화면 모두 세로 스크롤 목록에서 동일 카드 뼈대에 선택 컨트롤·우측 상태만 바뀐다

## 시각적 의도

> 이 패턴은 **왼쪽 선택 컨트롤 + 중앙 병원 정보 + (L3) 우측 상태**가 한 행에 정리된 카드처럼 보인다. 사용자의 시선은 L2에서는 체크 여부와 병원명, L3에서는 상태 배지와 거절 사유로 먼저 간다.

- **전체 인상**: 중간 밀도·구조적 border 카드. 한 화면에 여 병원을 스캔하기 쉽게 세로 리듬을 맞춘다
- **시각적 초점**: 병원명(subheading) + 적합 요약 1~2줄(body)
- **시각적 무게**: 선택 컨트롤 15% : 텍스트 55% : (L3) 상태·메타 30%
- **디자인 핵심**: L2·L3 **병원명·식별·요약 줄바꿈 규칙**을 통일해 동일 선택 풀 인식을 유지한다

---

## Visual Recipe

흰 카드 안에서 왼쪽 끝에만 체크(L2) 또는 라디오(L3)가 박히고, 그 옆으로 병원명이 한 단 굵게, 이어 거리·전문 한 줄이 작은 글씨로 붙는 전형적인 “병원 리스트” 행이다. L3에서는 우측 상단 또는 제목 옆에 작은 상태 칩이 붙어 승인·대기·거절을 즉시 구분하고, 거절 시에만 본문 아래에 사유 문장이 한 블록 더 생긴다. 그림자 없이 테두리만으로 카드가 나뉘어 현장 조명에서도 윤곽이 흐려지지 않으며, primary 색은 선택된 라디오·체크에만 얹혀 과도한 장식을 피한다.

## 레이아웃 구조

### 전체 프레임 `병원-행-카드`

| 속성 | 값 | 설계 의도 |
|------|-----|----------|
| 크기 | 390 × auto (최소 약 88px) | 1~2줄 요약 + (조건부) 사유 |
| 배경 | surface — white #FFFFFF (rgba(255,255,255,1)) | 목록 background #FAFAFA 대비 |
| 배치 | 수평 스택 | 선택 / 본문 / (L3) 상태 열 |
| 주축 정렬 | 시작 | 텍스트는 상단 기준 정렬 권장 |
| 교차축 정렬 | 시작 또는 중앙(한 줄 카드 시 중앙) | 체크·라디오는 세로 중앙 |
| 내부 여백 | 상16 하16 좌16 우16 — spacing.xl | 터치 여유 |
| 요소 간격 | 수평 12px — spacing.lg | 컨트롤과 본문 분리 |
| 모서리 | 10px — radius.lg | design-system 카드 반경 |
| 깊이 | 없음 | border 우선 |
| 테두리 | border — gray.300 #D5D7DA 1px | 카드 경계 |

### 구조 다이어그램

**L2 (체크)**

```
┌──────────────────────────────────────────────┐
│ ☐  │ 병원명 (subheading)                     │
│    │ 📍 거리 · 전문 요약 (caption/body)        │
│    │ 적합 요약 / 추천 사유 (body)             │
└──────────────────────────────────────────────┘
```

**L3 (라디오 + 상태)**

```
┌──────────────────────────────────────────────┐
│ ○  │ 병원명          [대기 중] [배지]          │
│    │ 거절 사유 블록 (조건부)                    │
│    │ 갱신 시각 (caption)                       │
└──────────────────────────────────────────────┘
```

## 시각 품질 검증

### Surface Depth

| 요소 | fill | 부모/배경 fill | 대비 | 조치 |
|------|------|--------------|------|------|
| 카드 | white | gray.50 | 충분 | — |
| 선택된 행 강조 (선택 정책) | surface-raised #F5F5F5 | white | 충분 | 전면 상태색 fill 금지 — 배경 틴트만 소면적 |

### Elevation

| 요소 | 유형 | shadow 토큰 | 근거 (visual-direction) |
|------|------|------------|----------------------|
| 병원 카드 | 목록 행 | 없음 | border 우선 |

### 타이포 위계

| 수준 | 역할 | 스펙 | 적용 요소 |
|------|------|------|----------|
| 1 | subheading | 18px/28px/0px/600 | 병원명 |
| 2 | body | 16px/24px/0px/400 | 적합 요약·사유 |
| 3 | caption | 12px/18px/0px/400 | 거리·시각 |

### 강조

- focal point: 병원명 + 적합 요약 — 타이포 위계로 결정 근거 전달
- L3: 상태 칩 — success/warning/error 역할 분리
- 근거: visual-direction.mdc > 색상 분배·역할 분리

### 밀도

- 행 전체 탭 영역: 최소 높이 88px 권장 — 44px 터치 ✅
- 체크·라디오 단독: 40×40 컨테이너 내 중앙 배치, 행 탭으로 보완
