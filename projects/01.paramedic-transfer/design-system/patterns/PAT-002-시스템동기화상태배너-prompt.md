# PAT-002 — 시스템·동기화 상태 배너

## 메타
- 등장 화면: G1, L1, L2, L3 (IA: 화면별 상단, 앱 바 아래 또는 대체 위치는 스펙 따름)
- 역할: **오프라인·동기화 중·실패·재시도** 등 네트워크·데이터 상태를 **한 줄**로 즉시 알리고, 필요 시 **재시도/안내** 동선을 연다
- 반복 구조: 동일 높이의 풀폭 스트립; 문구·아이콘·좌측 강조 띠·텍스트 링크만 상태별로 바뀜

## 시각적 의도

> 이 패턴은 **앱 바 바로 아래 얇은 정보 띠**처럼 보인다. 사용자의 시선은 **상태 아이콘 + 한 줄 메시지**로 먼저 가고, 문제가 해결 가능하면 **accent 색 재시도 링크**가 다음 초점이 된다.

- **전체 인상**: 조밀·기능적. 장식 배경 블록 없이 `surface-raised` 바탕 + 좌측 4px 상태 띠(visual-direction 허용 범위)
- **시각적 초점**: 상태 아이콘(의미) + 본문 한 줄(설명)
- **시각적 무게**: 아이콘+문구 85% : 액션 텍스트 15%
- **디자인 핵심**: 색만으로 구분하지 않음 — **아이콘 + 문구** 병행, 위험/경고는 토큰 색을 아이콘·좌측 띠에만 제한적 사용

---

## Visual Recipe

밝은 회색 한 톤(`surface-raised`)의 가로 띠가 화면 너비를 채우고, 왼쪽 끝에는 4px 세로 컬러 바가 상태 성격(경고·오류·성공·정보)을 짧게 찍어 준다. 그 옆에는 20px 안팎의 선명한 아이콘 한 개와 16px 본문 한 줄이 붙어, "오프라인입니다"·"동기화 중…"처럼 **읽고 바로 행동할 수 있는 문장**을 보여 준다. 재시도가 있으면 같은 줄 오른쪽 끝에 파란 링크 스타일(`accent`)으로 "다시 시도"만 얹어 CTA와 역할이 겹치지 않게 한다. 전체적으로 카드나 그림자 없이 **테두리 없음 또는 하단 1px만**으로 본문과 스무스하게 이어지게 할 수 있다.

## 레이아웃 구조

### 전체 프레임 `시스템·동기화 상태 배너`

| 속성 | 값 | 설계 의도 |
|------|-----|----------|
| 크기 | 390 × auto (최소 높이 약 44px, 내용 1줄 기준) | 터치·가독성; 조밀 유지 |
| 배경 | surface-raised — gray.100 #F5F5F5 (rgba(245,245,245,1)) | 본문 `surface`/`background`와 한 단계 구분 |
| 배치 | 수평 스택 | 아이콘 → 문구 → (옵션) trailing 액션 |
| 주축 정렬 | 시작(아이콘·텍스트) / 끝(trailing 링크) | 한 줄에 정보+액션 |
| 교차축 정렬 | 중앙 | 아이콘·텍스트 베이스라인 정렬 |
| 내부 여백 | 상 10 하 10 좌 12 우 12 — 좌측 4px 띠 별도 | spacing.xl(16px)과 근접; 세로는 44px 터치에 맞춤 |
| 요소 간격 | 아이콘–텍스트 12px | PAT-003과 동일 리듬(spacing.lg 계열) |
| 모서리 | 없음 | 풀블리드 스트립 |
| 깊이 | 없음 | shadow 미사용 |
| 테두리 | 없음 또는 하단만 border — gray.300 1px (화면 스펙 선택) | 본문과의 경계가 모호할 때만 하단 border |

### 구조 다이어그램

```
┃ ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░
┃  🔸  오프라인입니다. 연결 후 자동 동기화됩니다.     [다시 시도]
↑
4px 상태 띠 (error/warning/success/accent)
```

### 좌측 상태 띠 `status-rail`

| 속성 | 값 |
|------|-----|
| 크기 | 4 × (배너 높이 전체) px |
| 배경 | 상태별 — success.600 / warning.600 / error.600 / accent blue.600 (문서 「카드 좌측 상태 바」 규칙과 동일 너비) |
| 위치 | 프레임 최좌측, 배경과 플러시 |

### 본문 행 `banner-row`

#### 내부 요소

| 요소 | 유형 | 내용 | 타이포 | 색상 |
|------|------|------|--------|------|
| 상태 아이콘 | 아이콘 | 위 표 상태별 | — | 표의 색상 역할 |
| 메시지 | 텍스트 | 예: "오프라인입니다. 기록은 기기에 저장됩니다." | body — 16px / 24px / 0 / regular(400) | text-primary — gray.900 #181D27 |
| 보조 문구(선택) | 텍스트 | 같은 줄에 짧게 이어 쓰기(말줄임) | caption — 12px / 18px / 0 / 400 | text-secondary — gray.600 #535862 |
| 재시도 링크 | 텍스트 | "다시 시도" | body — 16px / 24px / 0 / medium(500) | accent — blue.600 #1570EF |

> 한 줄 초과 시: 메시지만 `caption`으로 내리거나 2줄 허용(스펙 선택). **패턴 구조(아이콘+문구+옵션 링크)는 유지.**

## 시각 품질 검증

### Surface Depth

| 요소 | fill | 부모/배경 fill | 대비 | 조치 |
|------|------|----------------|------|------|
| 배너 | gray.100 #F5F5F5 | white #FFFFFF 또는 gray.50 #FAFAFA | 충분 | visual-direction Surface 계층 표 참조 |
| 좌측 4px 띠 | 상태색 | gray.100 | 충분 | 면적 최소로 제한 |

### Elevation

| 요소 | 유형 | shadow 토큰 | 근거 (visual-direction) |
|------|------|------------|------------------------|
| 배너 | 인라인 스트립 | 없음 | 테두리·면 구분으로 처리, 플로팅 아님 |

### 타이포 위계

| 수준 | 역할 | 스펙 (크기/행간/자간/굵기) | 적용 요소 |
|------|------|---------------------------|----------|
| 1 | body | 16px / 24px / 0 / 400 | 주 메시지 |
| 2 | caption | 12px / 18px / 0 / 400 | 같은 줄 보조(선택) |
| 3 | body (강조 링크) | 16px / 24px / 0 / 500 | 재시도 — 색으로만 CTA와 구분 |

### 강조

- focal point: **아이콘 + 첫 문장** — 상태 인지
- 상태 표현: **아이콘 + 색(띠·아이콘) + 문구** 병행
- 근거: visual-direction.mdc > 색상 분배 규칙, 강조 방식

### 밀도

- 배너 행 최소 높이 **44px** 권장 — 탭 가능 영역(재시도 링크) 충족
- ❌ 미충족 시: 상하 padding 증가 또는 링크에 최소 44px 높이 히트 영역 확장
