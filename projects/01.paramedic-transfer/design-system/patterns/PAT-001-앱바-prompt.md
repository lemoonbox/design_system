# PAT-001 — 앱 바

## 메타
- 등장 화면: G1, L1, L2, L3, L4 (풀스크린 로컬), M1 (모달 헤더 변형)
- 역할: **뒤로/닫기**, **현재 화면 제목**, **우측 보조 액션(옵션)**을 한 줄 크롬으로 고정해 동선과 맥락을 잃지 않게 한다
- 반복 구조: 각 화면 상단(시스템 상태 아래)에 동일 높이·동일 토큰; M1만 닫기·타이틀 정렬 규칙이 다름

## 시각적 의도

> 이 패턴은 **흰색 상단 바에 얇은 하단 구분선**이 있는 모바일 표준 앱 바처럼 보인다. 사용자의 시선은 **중앙(또는 M1은 좌측) 화면 제목**으로 간 뒤, 필요 시 **뒤로/닫기**와 **우측 액션**을 순서대로 스캔한다.

- **전체 인상**: 조밀한 현장 UI에 맞춰 장식 없이 한 줄. 테두리로 본문과 단절(visual-direction: 테두리 우선)
- **시각적 초점**: 타이틀 — `heading` 스케일로 화면 목적이 즉시 읽힘
- **시각적 무게**: 타이틀 55% : 좌·우 터치 슬롯 45%(기능은 보조이나 44px 히트는 확보)
- **디자인 핵심**: 좌·우 아이콘은 `text-primary` 단색 스트로크로 통일, 제목만 굵은 위계

---

## Visual Recipe

페이지 배경(`background` gray.50) 위에 얇은 흰 `surface` 바가 붙고, 아래쪽만 1px `border`로 스크롤 영역과 경계가 선명해진다. 가운데(또는 모달에서는 왼쪽)에는 24px짜리 화면 제목이 한 줄로 잘리며(말줄임), 왼쪽에는 뒤로 꺾쇠가 담긴 정사각 터치 영역이, 오른쪽에는 필요할 때만 메뉴·필터 등 보조 아이콘이 같은 톤으로 배치된다. M1에서는 닫기(X)가 우측에 고정되고 타이틀은 모달 폭 기준으로 좌측 정렬해 **시트·다이얼로그** 맥락이 분명해진다. 전체적으로 브랜드 채색 없이 흑·회 계열 위계만 쓰고, **CTA 색은 하단/본문**에 남긴다.

## 레이아웃 구조

### 전체 프레임 `앱 바`

| 속성 | 값 | 설계 의도 |
|------|-----|----------|
| 크기 | 390 × 56 px (가로는 디바이스 폭 100%) | `heading` line-height 32px + 상하 여백으로 한 줄 수용 |
| 배경 | surface — white #FFFFFF (rgba(255,255,255,1)) | 본문 카드와 동일 표면 단계 |
| 배치 | 수평 한 줄, 3열(좌 슬롯 · 중앙 타이틀 · 우 슬롯) | 표준 모바일 내비 인지성 |
| 주축 정렬 | 양끝(좌·우 44 고정) + 가운데 영역 fill | 타이틀 중앙 정렬(풀스크린) |
| 교차축 정렬 | 중앙 | 아이콘·텍스트 수직 중앙 |
| 내부 여백 | 상 12 하 11 좌 0 우 0 — 하단 1px 보더 포함 시 시각 균형 | 좌우는 아이콘 슬롯 내부에서 중앙 정렬 |
| 요소 간격 | 타이틀과 좌우 슬롯 사이 최소 8px | 말줄임 시 충돌 방지 |
| 모서리 | 없음 | 상단 플러시 |
| 깊이 | 없음 | shadow 미사용(테두리로 분리, visual-direction) |
| 테두리 | 하단만 border — gray.300 #D5D7DA, border.width.default 1px | 크롬 vs 콘텐츠 구분 |

### 강조

- focal point: **타이틀** — `heading` 단일 위계, 색은 text-primary만
- 상태 표현: 앱 바에서 **상태색 사용 안 함**(PAT-002 배너로 이관)
- 근거: visual-direction.mdc > 강조 방식(크기+컬러는 CTA·상태 영역에 집중), 색상 분배 규칙

### 밀도

- 뒤로/닫기/우측 액션: **44×44 px** — 44px 터치 기준 충족
- 타이틀: 터치 대상 아님
