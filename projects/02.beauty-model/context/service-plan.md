# Service Plan — 승인형 예약 기반 뷰티모델 플랫폼 (MVP)

> 버전 0.1 · 작성일 2026-03-23 · 범위: MVP (홍대 헤어 단일 권역, 모바일 웹)

---

## 서비스 개요

- **목적**: 헤어 디자이너와 일반인 고객(모델 지원자) 간 채팅 없는 승인형 예약 플로우를 표준화하여, 포트폴리오용 시술 거래 성사 구조를 만든다
- **해결하는 문제**: 인스타그램 DM·오픈채팅·당근마켓 등 비정형 채널에 의존하던 뷰티모델 모집의 핵심 문제는 수요 부족이 아니라 **거래 성사 구조의 부재**. 채팅 조율 피로, 노쇼, 허위지원, 사진·동의 개별 관리 부담이 공급자(디자이너)의 핵심 불편이며, 고객 측에서는 모집 정보 분산·예약 과정 불투명·조건 불명확이 이탈을 유발한다
- **MVP 검증 목표**: "홍대 헤어 디자이너의 빈 슬롯을, 채팅 없이, 실제 예약으로 전환할 수 있는가"

## 사용자

### 공급자 — 헤어 디자이너
- **주요 사용자**: 홍대 활동 인턴·주니어·1인 헤어 디자이너
- **사용 맥락**: 포트폴리오 사진·SNS 콘텐츠 확보를 위해 빈 시술 슬롯에 뷰티모델 모집이 필요할 때, 스마트폰으로 상시 관리
- **핵심 행동**:
  1. 시술 상품(모집글) 등록 및 조건 설정
  2. 신청자 프로필·사진 확인 후 승인/거절/대안 일정 제안
  3. 예약 확정 및 시술 완료 처리

### 수요자 — 일반 고객 (모델 지원자)
- **주요 사용자**: 홍대 생활권 20대, 합리적 가격 시술을 원하는 일반인 (전문 모델 경험 불필요)
- **사용 맥락**: 저렴한 가격(남성 커트 5천원, 여성 커트 1만원 수준)에 시술받고자 할 때, 스마트폰으로 탐색 및 신청
- **핵심 행동**:
  1. 상품 탐색 및 조건 확인
  2. 희망 일정 선택·사진 제출·동의 체크 후 신청
  3. 승인 알림 수신 후 결제 및 예약 확정

### 운영자
- 플랫폼 운영팀. 거래 상태 관리, 분쟁 처리, 지표 모니터링 담당

## 기능 범위

### 필수 기능 (In Scope)

| 영역 | 기능 |
|------|------|
| 인증 | 디자이너·고객 휴대폰 본인인증, 로그인/로그아웃 |
| 디자이너 | 프로필 등록, 시술 상품 등록·수정·삭제, 모집글 관리 |
| 고객 | 프로필 등록, 사진 등록·관리 |
| 탐색 | 시술 상품 목록 조회, 상품 상세 조회 |
| 예약 플로우 | 신청 → 사진 제출 → 승인/거절 → 대안 일정 제안 → 결제/예약금 → 확정 |
| 알림 | 상태 변경 시 앱 푸시(Web Push) + SMS 알림 |
| 동의 관리 | 초상권·후기·SNS 활용 동의 저장 (DB 영구 보존) |
| 취소·노쇼 | 취소 규정 안내, 노쇼 기록 |
| 운영자 | 예약 상태 관리, 디자이너·고객 계정 관리, 지표 대시보드 |

### 선택 기능 (P1 — MVP 포함 가능)

- 신청 후 48h 디자이너 미응답 시 자동 거절 처리
- 결제 후 24h 미결제 시 자동 취소 처리
- 대안 일정 제안 후 24h 고객 미응답 시 자동 취소
- 모집 인원 도달 시 상품 자동 마감
- 예약 24h 전 리마인더 알림
- 노쇼 3회 초과 시 신청 제한 플래그 (운영자 검토)
- 시술 참고 이미지 첨부 (최대 3장)

### 제외 (Out of Scope — MVP)

- 자유 채팅 (채팅 배제가 핵심 차별점)
- 커뮤니티·피드
- AI 추천·매칭
- 리뷰·평점 시스템
- 디자이너 유료 멤버십
- 전국 서비스·카테고리 확장
- 네이티브 앱 (iOS/Android)
- SNS 소셜 로그인

## 비즈니스 목표

- **목표**: 홍대 헤어 권역에서 채팅 없는 승인형 예약 거래 성사 구조 검증
- **성공 지표**:

| 지표 | 목표값 |
|------|--------|
| 활성 디자이너 수 | 10명 이상 (모집글 1건 이상 등록) |
| 고객 신청 건수 | 100건 이상 (사진 제출 포함 완료 기준) |
| 완료 예약률 | 40~50% 이상 |
| 노쇼율 | 10% 이하 |
| 채팅 없이 처리된 비율 | 70% 이상 |
| 디자이너 재등록률 | 50% 이상 |
| 고객 재신청률 | 15% 이상 |

- **검증 가설**:
  - H1: 홍대 디자이너는 채팅형보다 승인형 예약 방식을 선호한다 → 재등록률 50% 이상
  - H2: 고객은 일반인 헤어모델 + 저가 커트 + 승인형 예약에 실제로 신청한다 → 신청 완료율 50% 이상
  - H3: 홍대 1개 권역 안에서 반복 거래가 쌓인다 → 고객 재신청률 15% 이상

## 비즈니스 로직

### 수수료
```
수수료 = MAX(시술비 × 15%, 5,000원)
디자이너 정산액 = 시술비 - 수수료
정산 시점: 시술 완료 처리 후 T+3 영업일
```

### 예약금
```
예약금 = 시술비 × 30% (디자이너가 활성화 여부 설정)
전액 결제 시 예약금 불필요
노쇼 발생 시 예약금 → 디자이너 귀속 (플랫폼 수수료 차감)
```

### 취소·환불
```
시술 48시간 전 취소: 전액 환불
시술 48시간 이내 취소: 예약금 미환불
```

## 제약 조건

- **플랫폼**: 모바일 웹 (PWA 고려, 360px~430px 최적화), iOS Safari 15+ / Android Chrome 100+
- **권역**: MVP는 홍대 헤어 단일 권역
- **연동 시스템**:
  - 결제 PG: 토스페이먼츠 또는 포트원(아임포트) — 착수 전 결정 필요
  - SMS: 솔라피 / 알리고 / 카카오 알림톡 — 착수 전 결정 필요
  - 사진 저장소: AWS S3 또는 Cloudflare R2
- **보안**: JWT + Refresh Token, HTTPS 필수, 업로드 사진 서명된 URL로만 접근, 카드정보 서버 미저장
- **브랜드/규정**: 플랫폼명 미정 (OI-07). 초상권·후기·SNS 동의 내용 DB 영구 보존 (법적 근거)
