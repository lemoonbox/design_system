## IA 구조

```
[공통]
- [A1] 온보딩/스플래시
  - [A2] 로그인
  - [A3] 회원가입 (스텝 플로우: 본인인증 → 역할 선택 → 프로필)
    → 완료 후 역할에 따라 고객/디자이너 앱으로 분기

[고객 앱]
하단 탭: 탐색(CG1) / 예약(CG2) / 알림(CG3) / MY(CG4)

- [CG1] 탐색 (상품 목록)
  - [CL1] 상품 상세
    - [CL2] 신청하기
      - [CM2] 동의 내역 확인 (Modal)
      → 신청 완료 후 CG2로 이동
    - [CL3] 결제 (디자이너 승인 후 진입)

- [CG2] 내 예약
  - [CL4] 예약 상세
    - [CM1] 취소 확인 (Modal)
    - [CL3] 결제 (대기 → 승인 상태에서 재진입)

- [CG3] 알림

- [CG4] MY
  - [CL5] 프로필 편집
  - [CL6] 내 사진 관리

[디자이너 앱]
하단 탭: 모집글(DG1) / 신청관리(DG2) / 예약(DG3) / MY(DG4)

- [DG1] 모집글 목록
  - [DL1] 모집글 상세
    - [DL2] 모집글 등록/편집

- [DG2] 신청 관리
  - [DL3] 신청자 상세
    - [DM1] 대안 일정 제안 (Modal)

- [DG3] 예약 목록
  - [DL4] 예약 상세
    - [DM2] 시술 완료 확인 (Modal)

- [DG4] MY
  - [DL5] 프로필 편집
```

## 서비스 플로우

### 플로우 1. 고객 신청 → 승인 → 결제 (핵심 happy path)

```
[A1] 온보딩
  → [A2] 로그인 (신규: A3 회원가입 후 진입)
  → [CG1] 탐색 (상품 목록)
  → [CL1] 상품 상세 확인
  → [CL2] 신청하기 (일정 선택 + 사진 제출 + 동의 체크)
    → [CM2] 동의 내역 확인 (Modal, 선택)
  → 신청 완료 → [CG2] 내 예약 (대기 중 상태)
  ↓ (디자이너가 DL3에서 승인)
  → 푸시/SMS 알림 수신 → [CG3] 알림 또는 [CG2] 예약
  → [CL4] 예약 상세 → '결제하기' 버튼
  → [CL3] 결제 완료
  → [CG2] 예약 확정 상태
```

### 플로우 2. 디자이너 신청 처리

```
[DG2] 신청 관리 (대기 뱃지 표시)
  → [DL3] 신청자 상세 (프로필·사진 열람)
  → 승인 탭
    → 고객 알림 발송 (푸시 + SMS)
    → [DG3] 예약 목록에 등록
  (또는)
  → 거절 탭
    → 거절 사유 선택 후 발송
    → 고객 알림 발송
  (또는)
  → [DM1] 대안 일정 제안 (Modal)
    → 고객 수락 → [DG3] 예약 등록
    → 고객 거절 / 24h 무응답 → 자동 취소
```

### 플로우 3. 예외 — 취소 (48h 이내)

```
[CG2] 내 예약
  → [CL4] 예약 상세
  → '예약 취소' 버튼
  → [CM1] 취소 확인 (예약금 미환불 규정 고지)
  → 확인
  → 예약 취소 완료 → [CG2] 취소 상태 표시
  → 디자이너 알림 발송
```

### 플로우 4. 예외 — 신청 실패 (필수 항목 미입력)

```
[CL2] 신청하기
  → 사진 미첨부 또는 동의 미체크 상태로 제출 시도
  → 인라인 오류 표시 (화면 유지)
  → 수정 후 재제출
  → 신청 완료
```

### 플로우 5. 시술 완료 처리 및 정산

```
[DG3] 예약 목록
  → [DL4] 예약 상세
  → '시술 완료' 버튼
  → [DM2] 시술 완료 확인 (Modal)
    → 동의 내역 자동 확인 (저장 완료 여부)
  → 완료 처리
  → 정산 스케줄 생성 (시술 완료 T+3 영업일)
  → 고객 완료 알림 발송
```
