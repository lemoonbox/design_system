# A1 — 온보딩/스플래시

## 개요
- 목적: 앱에 최초 진입하거나 로그아웃 후 재진입할 때 서비스를 소개하고 로그인·회원가입으로 분기한다
- 진입 경로: 앱 URL 최초 접근 / 로그아웃 후 리다이렉트 / 세션 만료 후 리다이렉트
- 이탈 경로:
  - '로그인' 버튼 탭 → A2
  - '회원가입' 버튼 탭 → A3
  - 유효 토큰 자동 감지 → 역할에 따라 CG1(고객) 또는 DG1(디자이너) 자동 이동

---

## 레이아웃 구조
- 네비게이션: 없음 (전체화면)
- 헤더: 없음
- 주요 영역: 단일 콘텐츠 (브랜드 로고 + 서비스 슬로건)
- 하단 고정 영역: 있음 — '로그인' 버튼 (Primary) + '회원가입' 버튼 (Secondary)

---

| 필드명 | 타입 | 필수 여부 | 표시 방식 | 비고 |
|--------|------|----------|----------|------|

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 로그인 진입 | '로그인' 버튼 탭 | A2로 이동 | — |
| 회원가입 진입 | '회원가입' 버튼 탭 | A3으로 이동 | — |
| 자동 로그인 | 화면 마운트 시 토큰 유효성 검사 | 토큰 유효 → 역할에 따라 CG1/DG1으로 자동 이동 | Refresh Token 유효 시 |

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 토큰 없음 또는 만료 | 로고 + 슬로건 + 하단 버튼 2종 표시 |
| 자동 로그인 처리 중 | 토큰 유효성 확인 중 | 로딩 스피너 (버튼 비표시), 완료 후 자동 이동 |

---

## 엣지케이스
- **Refresh Token 만료**: 토큰 재발급 실패 시 기본 상태로 폴백 (로그인/회원가입 버튼 표시)
- **PWA 오프라인 진입**: 네트워크 없음 상태에서는 자동 로그인 시도 생략, 기본 상태 표시 + 온라인 복귀 시 재시도
