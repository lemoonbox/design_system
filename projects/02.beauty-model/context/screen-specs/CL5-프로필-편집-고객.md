# CL5 — 프로필 편집 (고객)

## 개요
- 목적: 고객이 자신의 기본 프로필 정보를 수정한다
- 진입 경로: CG4(MY) → '프로필 편집' 메뉴 탭
- 이탈 경로:
  - 저장 완료 → CG4(MY)
  - 뒤로가기 (변경사항 있을 경우 확인 Alert) → CG4

---

## 레이아웃 구조
- 네비게이션: 상단 바 (뒤로가기 버튼 + "프로필 편집" 타이틀)
- 헤더: 없음
- 주요 영역: 폼 (필드 목록)
- 하단 고정 영역: 있음 — '저장' CTA 버튼

---

## 데이터 필드

| 필드명 | 타입 | 필수 여부 | 표시 방식 | 비고 |
|--------|------|----------|----------|------|
| 이름 | text | Y | 텍스트 입력 필드 | 최대 20자 |
| 성별 | select | N | 세그먼트 선택 (남 / 여 / 선택 안 함) | — |
| 휴대폰 번호 변경 | text | N | 텍스트 + '변경' 버튼 | 변경 시 인증번호 재발송 플로우 진행 (A2와 동일 구조) |

---

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 저장 | '저장' 버튼 탭 | 프로필 업데이트 → CG4로 이동 + 성공 토스트 | 필수 필드 유효 |
| 저장 실패 (검증) | '저장' 버튼 탭 | 인라인 오류 표시 | 이름 비어있음 |
| 번호 변경 | '변경' 버튼 탭 | 인증번호 발송 플로우 인라인 표시 (A2 폼 삽입) | — |
| 뒤로가기 | 상단 뒤로가기 탭 | 변경사항 있으면 "저장하지 않고 나가시겠습니까?" Alert | — |

---

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 기존 데이터 로드 완료 | 현재 값 채워진 폼 |
| 수정 중 | 사용자 입력 중 | '저장' 버튼 활성 |
| 저장 중 | API 호출 중 | CTA 버튼 로딩 |
| 로딩 | 기존 데이터 조회 중 | 폼 스켈레톤 |
