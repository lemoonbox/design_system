# CM1 — 취소 확인 (Modal)

## 개요
- 목적: 예약 취소 전 취소 규정을 고지하고 고객의 최종 의사를 확인한다
- 진입 경로: CL4(예약 상세) → '예약 취소' 버튼 탭
- 이탈 경로:
  - '취소 확인' 버튼 탭 → Modal 닫힘 → CG2(내 예약, 취소 상태)
  - '닫기' 또는 배경 탭 → Modal 닫힘 → CL4(예약 상세, 변경 없음)

---

## 레이아웃 구조
- 네비게이션: 없음 (Modal)
- 헤더: 있음 — "예약을 취소하시겠습니까?" 제목
- 주요 영역: 취소 규정 고지 텍스트
- 하단 고정 영역: 버튼 2종 ('취소 확인' Primary + '돌아가기' Secondary)

---

## 데이터 필드

| 필드명 | 타입 | 필수 여부 | 표시 방식 | 비고 |
|--------|------|----------|----------|------|
| 취소 규정 안내 | text | Y | 본문 텍스트 | 48시간 기준 분기 표시 (아래 참고) |
| 환불 예정 금액 | number | N | 강조 텍스트 | 결제 완료 상태에서만 표시 |

---

## 취소 규정 분기 표시

| 시나리오 | 표시 내용 |
|---------|----------|
| 시술 48시간 초과 남은 경우 | "시술 48시간 전 취소로 전액 환불됩니다." |
| 시술 48시간 이내인 경우 | "시술 48시간 이내 취소입니다. 예약금은 환불되지 않습니다." (경고 강조) |
| 결제 전 취소 (예약금 미결제) | "결제 전 취소로 별도 환불 없이 신청이 취소됩니다." |

---

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 취소 확인 | '취소 확인' 버튼 탭 | 예약 취소 API 호출 → 성공 시 Modal 닫힘 → CG2(취소 상태) | — |
| 취소 처리 중 | API 호출 중 | '취소 확인' 버튼 로딩 | — |
| 돌아가기 | '돌아가기' 버튼 탭 또는 배경 탭 | Modal 닫힘 → CL4 유지 | — |

---

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | Modal 열림 | 규정 텍스트 + 버튼 2종 표시 |
| 처리 중 | API 호출 중 | 확인 버튼 로딩, 배경 탭 비활성 |
| 오류 | 취소 API 실패 | 토스트 오류 + Modal 유지 |

## 엣지케이스
- **이미 완료·취소된 예약에서 취소 시도**: 서버 409 응답 → "이미 처리된 예약입니다" 안내 후 Modal 닫힘 → CG2 새로고침

---

## 패턴 후보
- **규정 고지 Modal 구조**: DM2(시술 완료 확인)와 동일한 2-버튼 확인 Modal 레이아웃
