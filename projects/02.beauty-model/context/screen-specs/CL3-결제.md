# CL3 — 결제

## 개요
- 목적: 고객이 디자이너 승인 후 예약금 또는 전액을 결제하여 예약을 확정한다
- 진입 경로: CL4(예약 상세) → '결제하기' 버튼 탭 (디자이너 승인 완료 상태에서만 진입 가능)
- 이탈 경로:
  - 결제 완료 → CG2(내 예약, 확정 상태)
  - PG창 닫기(취소) → CL3 유지 (결제 미완료 상태)
  - 뒤로가기 → CL4(예약 상세)

---

## 레이아웃 구조
- 네비게이션: 상단 바 (뒤로가기 버튼 + 화면 제목 "결제")
- 헤더: 있음 — 시술명, 디자이너명, 확정 일정 요약 카드
- 주요 영역: 단일 콘텐츠 (결제 금액 상세 + 결제 수단 선택)
- 하단 고정 영역: 있음 — '결제하기' CTA 버튼 (금액 표시 포함)

## 엣지케이스
- **결제 전 모집글 삭제**: 결제 API 호출 시 서버 검증 실패 → "예약을 진행할 수 없습니다" 안내 후 CG2로 이동
- **중복 결제 방지**: 서버에서 예약 건당 1회 결제만 허용; 중복 요청 시 409 응답 → 이미 완료된 결제 상태 표시
- **PG 연동 오류 (네트워크 단절)**: 타임아웃 발생 시 결제 상태 불확실 → "결제 상태를 확인 중입니다" 안내 + 서버 side 주문 조회로 최종 상태 확인 후 처리
- **예약금 설정 없는 상품 (전액 결제)**: '예약금 30%' 옵션 미표시, 전액만 표시
- **세션 만료 후 결제 시도**: PG 콜백 처리 중 세션 만료 → 재로그인 후 CG2에서 결제 상태 확인 안내

## 패턴 후보
- **시술 요약 카드**: CL2(신청하기), CL4(예약 상세) 헤더와 동일 구조 — 공통 패턴화 대상
- **금액 상세 섹션**: 항목·금액 행 나열 레이아웃 — 정산 내역(DG4) 화면과 유사 구조
- **규정 펼침 컴포넌트**: 취소·환불 규정 접힘/펼침 — CM1(취소 확인) Modal에서도 동일 규정 텍스트 재사용
