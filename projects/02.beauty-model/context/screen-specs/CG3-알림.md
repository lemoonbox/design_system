# CG3 — 알림

## 개요
- 목적: 예약 상태 변경 알림을 시간순으로 확인하고 해당 화면으로 빠르게 이동한다
- 진입 경로: 하단 탭 '알림' / 푸시 알림 탭 → 앱 열기 후 자동 진입
- 이탈 경로:
  - 알림 항목 탭 → 관련 화면 (CL4 예약 상세 또는 CG2 내 예약)

---

## 레이아웃 구조
- 네비게이션: 하단 탭 ('알림' 활성)
- 헤더: 있음 — "알림" 타이틀 + '모두 읽음' 텍스트 버튼 (우측)
- 주요 영역: 알림 항목 목록 (최신순 수직 스크롤)
- 하단 고정 영역: 없음

---

## 데이터 필드 (알림 항목 단위)

| 필드명 | 타입 | 필수 여부 | 표시 방식 | 비고 |
|--------|------|----------|----------|------|
| 알림 유형 | select | Y | 아이콘 또는 컬러 도트 | 승인 / 거절 / 대안 제안 / 결제 요청 / 확정 / 완료 / 취소 |
| 알림 본문 | text | Y | 본문 텍스트 | 예: "OOO 디자이너가 신청을 승인했습니다" |
| 수신 시각 | datetime | Y | 서브텍스트 | "방금 전 / 3분 전 / 2시간 전 / 3/22" 형식 |
| 읽음 여부 | boolean | Y | 미읽음 시 배경 강조 또는 도트 | 탭 시 읽음 처리 |

## 알림 유형 목록

| 알림 유형 | 본문 예시 | 이동 대상 |
|----------|----------|----------|
| 승인됨 | "OOO 디자이너가 신청을 승인했습니다. 결제를 완료해 주세요." | CL4 (해당 예약) |
| 거절됨 | "OOO 디자이너가 신청을 거절했습니다." | CL4 (해당 예약) |
| 대안 일정 제안 | "OOO 디자이너가 다른 일정을 제안했습니다." | CL4 (해당 예약) |
| 결제 완료 | "결제가 완료되었습니다. 예약이 확정되었습니다." | CL4 (해당 예약) |
| 예약 취소 | "예약이 취소되었습니다." | CG2 |
| 시술 완료 | "시술이 완료 처리되었습니다." | CL4 (해당 예약) |
| 리마인더 | "내일 OOO 시술 예약이 있습니다." (P1) | CL4 (해당 예약) |

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 알림 상세 이동 | 알림 항목 탭 | 유형별 이동 대상 화면으로 이동 + 읽음 처리 | — |
| 모두 읽음 | '모두 읽음' 버튼 탭 | 전체 알림 읽음 처리 | 미읽음 알림 존재 시 활성 |

---

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 알림 1건 이상 | 목록 표시 |
| 로딩 | 조회 중 | 항목 스켈레톤 |
| 빈 상태 | 알림 없음 | "아직 알림이 없습니다" 안내 |
| 오류 | 네트워크 실패 | 오류 메시지 + 재시도 버튼 |

## 엣지케이스
- **알림 탭 시 연결된 예약이 삭제된 경우**: 탭 시 "더 이상 존재하지 않는 예약입니다" 토스트 + CG2로 이동
