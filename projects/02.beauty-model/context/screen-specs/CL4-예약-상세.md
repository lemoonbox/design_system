# CL4 — 예약 상세

## 개요
- 목적: 개별 예약의 현재 상태와 상세 정보를 확인하고 상태에 따른 액션(결제·취소)을 수행한다
- 진입 경로: CG2(내 예약) → 예약 카드 탭
- 이탈 경로:
  - '결제하기' 버튼 탭 → CL3(결제)
  - '예약 취소' 버튼 탭 → CM1(취소 확인 Modal)
  - CM1 취소 확인 완료 → CG2(취소 상태)
  - 뒤로가기 → CG2

---

## 레이아웃 구조
- 네비게이션: 상단 바 (뒤로가기 버튼 + 화면 제목 "예약 상세")
- 헤더: 있음 — 예약 상태 배지 (대형)
- 주요 영역: 단일 콘텐츠 스크롤 (시술 정보 → 일정 → 결제 정보 → 취소 규정)
- 하단 고정 영역: 있음 — 상태별 CTA 버튼 (아래 액션 참고)

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 결제하기 | '결제하기' CTA 탭 | CL3으로 이동 | 승인됨 상태 |

| 뒤로가기 | 상단 바 뒤로가기 탭 | CG2로 이동 | — |

## 상태별 하단 CTA

| 예약 상태 | 하단 CTA |
|----------|---------|
| 대기 중 | '예약 취소' 버튼 (Secondary) |
| 승인됨 | '결제하기' (Primary) + '예약 취소' (Secondary) |
| 확정 | '예약 취소' (Secondary) |
| 완료 | 없음 (또는 "시술이 완료되었습니다" 안내) |
| 취소 | 없음 (또는 "취소된 예약입니다" 안내) |
| 거절 | 없음 (거절 사유 표시) |

## 엣지케이스
- **시술 당일 취소 시도**: 취소 버튼은 표시되나 CM1에서 예약금 미환불 명시 후 진행
- **완료된 예약 취소 시도**: CTA 버튼 미표시 (서버 단에서도 취소 불가 처리)
- **승인됨 상태에서 결제 만료**: 페이지 재진입 시 자동 취소 상태로 반영 (P1)
