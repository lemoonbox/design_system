# A3 — 회원가입

## 개요
- 목적: 본인인증 → 역할 선택 → 프로필 초기설정 3단계 스텝 플로우로 신규 계정을 생성한다
- 진입 경로: A1 '회원가입' 버튼 / A2 '회원가입' 링크
- 이탈 경로:
  - 가입 완료 → 역할에 따라 CG1(고객) 또는 DG1(디자이너)
  - 각 스텝 뒤로가기 → 이전 스텝 (Step 1에서 뒤로가기 → A1)

---

## 레이아웃 구조
- 네비게이션: 상단 바 (뒤로가기 + 현재 스텝 표시)
- 헤더: 있음 — 스텝 진행 바 (1/3 → 2/3 → 3/3) + 스텝 제목
- 주요 영역: 폼 (스텝별 전환)
- 하단 고정 영역: 있음 — '다음' CTA 버튼 (마지막 스텝: '가입 완료')

---

## 스텝 구성

### Step 1. 본인인증
A2(로그인)의 휴대폰 인증과 동일한 폼 구조를 사용한다.

| 필드명 | 타입 | 필수 여부 | 비고 |
|--------|------|----------|------|
| 휴대폰 번호 | text (숫자) | Y | 이미 가입된 번호 입력 시 오류 |
| 인증번호 | text (숫자 6자리) | Y | 유효시간 3분 |

### Step 2. 역할 선택

| 필드명 | 타입 | 필수 여부 | 비고 |
|--------|------|----------|------|
| 역할 | select | Y | 고객(모델 지원자) / 디자이너 — 카드 선택 UI; 가입 후 변경 불가 안내 |

### Step 3. 프로필 초기설정 (역할별 분기)

**공통 필드**

| 필드명 | 타입 | 필수 여부 | 비고 |
|--------|------|----------|------|
| 이름 | text | Y | 실명 권장 안내 |
| 서비스 이용약관 동의 | boolean | Y | 전문 보기 링크 포함 |
| 개인정보 처리방침 동의 | boolean | Y | 전문 보기 링크 포함 |

**고객 추가 필드**

| 필드명 | 타입 | 필수 여부 | 비고 |
|--------|------|----------|------|
| 성별 | select | N | 남 / 여 / 선택 안 함; 디자이너가 모집 조건 필터에 사용 |

**디자이너 추가 필드**

| 필드명 | 타입 | 필수 여부 | 비고 |
|--------|------|----------|------|
| 살롱/작업 공간 주소 | text | Y | 홍대 권역 확인용; 추후 지도 연동 |
| 디자이너 소개 | text | N | 최대 100자; 추후 DL5에서 수정 가능 |

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 다음 | '다음' 버튼 탭 | 다음 스텝으로 전환 | 현재 스텝 필수 필드 완료 |
| 가입 완료 | '가입 완료' 버튼 탭 (Step 3) | 계정 생성 → CG1/DG1으로 이동 | 전체 필수 필드 완료 + 약관 동의 |
| 뒤로가기 | 상단 바 뒤로가기 탭 | 이전 스텝으로 이동 (Step 1에서는 A1으로) | — |
| 약관 전문 보기 | 약관 링크 탭 | 약관 텍스트 Modal 또는 별도 페이지 | — |

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 스텝 진행 | 각 스텝 활성 | 진행 바 업데이트, 해당 스텝 폼 표시 |
| 필드 검증 오류 | 필수 미입력 또는 형식 오류 | 인라인 오류 메시지 |
| 제출 중 | 가입 API 호출 중 | CTA 버튼 로딩 |
| 오류 | 서버 오류 | 토스트 오류 메시지 + 재시도 유도 |

## 엣지케이스
- **이미 가입된 번호로 Step 1 시도**: 409 응답 → "이미 가입된 번호입니다. 로그인 하시겠습니까?" + A2 링크
- **Step 3 제출 중 세션 만료**: 재인증 안내 후 Step 1부터 재진행 (입력 데이터 최대한 보존)
- **역할 선택 Step 2에서 뒤로가기**: Step 1로 복귀 (인증 번호 재입력 불필요, 인증 상태 유지)

---

## 패턴 후보
- **휴대폰 인증 폼**: A2(로그인)과 동일 구조 — 공통 패턴화 대상
- **역할 선택 카드 UI**: 2종 선택 카드 — 다른 화면에서 재사용 가능성 낮음
