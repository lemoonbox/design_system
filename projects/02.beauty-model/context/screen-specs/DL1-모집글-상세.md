# DL1 — 모집글 상세

## 개요
- 목적: 디자이너가 자신의 모집글 내용을 확인하고 편집·마감·신청 관리로 이동한다
- 진입 경로: DG1(모집글 목록) → 모집글 카드 탭
- 이탈 경로:
  - '편집' 버튼 탭 → DL2(편집 모드)
  - '신청 관리' 버튼 탭 → DG2(신청 관리, 해당 모집글 필터)
  - 뒤로가기 → DG1

---

## 레이아웃 구조
- 네비게이션: 상단 바 (뒤로가기 + "모집글 상세" 타이틀 + 우측 '편집' 텍스트 버튼)
- 헤더: 없음 (참고 이미지가 상단을 차지)
- 주요 영역: 스크롤 (참고 이미지 → 시술 정보 → 모집 현황 → 일정 목록)
- 하단 고정 영역: 있음 — '신청 관리' 버튼 (대기 수 뱃지 포함) + 모집 상태 전환 버튼

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 편집 | '편집' 버튼 탭 | DL2(편집 모드)로 이동 | 마감 상태가 아닌 경우 |
| 신청 관리 이동 | '신청 관리' 버튼 탭 | DG2로 이동 (해당 모집글 필터 적용) | — |
| 모집 마감 | '마감 처리' 버튼 탭 | 마감 확인 Alert → 확인 → 모집 상태 '마감' 전환 | 모집 중 상태 |
| 모집 재개 | '모집 재개' 버튼 탭 | 모집 상태 '모집 중' 전환 | 마감 상태 + 잔여 일정 있음 |
| 모집글 삭제 | 편집 화면(DL2)에서 진행 | — | DL2에서 처리 |
| 뒤로가기 | 상단 바 뒤로가기 탭 | DG1으로 이동 | — |

---

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 (모집 중) | 잔여 인원 있음 | 모든 액션 버튼 활성 |
| 마감 | 인원 소진 또는 수동 마감 | '편집' 제한, '모집 재개' 버튼 표시 |
| 임시저장 | 저장 미완료 상태 | '임시저장' 배지 + '편집 완료' 유도 |
| 로딩 | 조회 중 | 스켈레톤 |
| 오류 | 네트워크 실패 | 오류 메시지 + 재시도 |

## 엣지케이스
- **삭제된 모집글 진입**: 서버 404 → "삭제된 모집글입니다" 안내 + DG1 이동
- **임시저장 상태 모집글**: 고객에게 노출되지 않음; DL1에서 편집 완료 후 '게시' 처리
