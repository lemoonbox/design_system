# CG2 — 내 예약

## 개요
- 목적: 고객이 신청부터 완료까지 모든 예약의 현재 상태를 한눈에 확인한다
- 진입 경로: 하단 탭 '예약' / CL2(신청 완료) 후 자동 이동 / CL3(결제 완료) 후 자동 이동
- 이탈 경로:
  - 예약 카드 탭 → CL4(예약 상세)

---

## 레이아웃 구조
- 네비게이션: 하단 탭 ('예약' 활성)
- 헤더: 있음 — "내 예약" 타이틀
- 주요 영역: 상태 필터 탭 + 예약 카드 목록 (수직 스크롤)
- 하단 고정 영역: 없음 (하단 탭이 고정)

---

## 데이터 필드 (예약 카드 단위)

| 필드명 | 타입 | 필수 여부 | 표시 방식 | 비고 |
|--------|------|----------|----------|------|
| 예약 상태 | select | Y | 컬러 배지 | 대기 중 / 승인됨 (결제 필요) / 확정 / 완료 / 취소 / 거절 |
| 시술명 | text | Y | 카드 제목 | — |
| 디자이너명 | text | Y | 카드 서브텍스트 | — |
| 시술 일정 | date+time | Y | 카드 본문 | — |
| 결제 금액 | number | N | 카드 하단 | 결제 완료 상태 이후만 표시 |
| 결제 마감 카운트다운 | timer | N | 카드 내 강조 텍스트 | 승인됨(결제 필요) 상태에서만 표시; 24h 만료 (P1) |

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 예약 1건 이상 | 카드 목록 표시 (최신순) |
| 로딩 | 데이터 조회 중 | 카드 스켈레톤 |
| 빈 상태 (전체 탭) | 신청 이력 없음 | "아직 신청한 시술이 없습니다" + CG1 이동 버튼 |
| 빈 상태 (개별 탭) | 해당 상태 예약 없음 | "해당 상태의 예약이 없습니다" |
| 오류 | 네트워크 실패 | 오류 메시지 + 재시도 버튼 |

---

## 엣지케이스
- **승인됨(결제 필요) 상태에서 24h 초과**: 카운트다운 만료 후 자동 취소 처리 (P1) → 상태가 '취소'로 전환, 토스트 알림 표시
- **여러 건의 결제 대기 중 동시 존재**: 모두 '진행 중' 탭에 표시, 카드별 독립 카운트다운
