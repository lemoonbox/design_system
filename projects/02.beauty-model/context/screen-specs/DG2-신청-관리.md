# DG2 — 신청 관리

## 개요
- 목적: 디자이너가 대기 중인 신청자를 모집글별로 확인하고 빠르게 처리한다
- 진입 경로: 하단 탭 '신청관리' / DL1(모집글 상세) → '신청 관리' 버튼 탭 (해당 모집글 필터 적용)
- 이탈 경로:
  - 신청자 카드 탭 → DL3(신청자 상세)

---

## 레이아웃 구조
- 네비게이션: 하단 탭 ('신청관리' 활성)
- 헤더: 있음 — "신청 관리" 타이틀
- 주요 영역: 모집글별 섹션 헤더 + 신청자 카드 목록 (또는 모집글별 탭 전환)
- 하단 고정 영역: 없음

---

## 데이터 필드 (신청자 카드 단위)

| 필드명 | 타입 | 필수 여부 | 표시 방식 | 비고 |
|--------|------|----------|----------|------|
| 신청자 이름 | text | Y | 카드 제목 | — |
| 신청자 프로필 사진 | image | N | 아바타 썸네일 | 없으면 기본 아바타 |
| 희망 시술 일정 | date+time | Y | 카드 본문 | — |
| 신청 일시 | datetime | Y | 카드 서브텍스트 | "방금 전 / 3시간 전 / 3/22" |
| 신청 처리 상태 | select | Y | 배지 (대기 중 / 승인 / 거절 / 대안 제안 중) | — |
| 신규 여부 | boolean | N | 미읽음 도트 또는 'NEW' 뱃지 | 마지막 접속 이후 새로 들어온 신청 |

---

## 모집글별 섹션 구조

- 섹션 헤더: 모집글명 + 대기 중 신청 수 + 전체 신청 수
- 섹션 하위: 해당 모집글의 신청자 카드 목록 (대기 중 → 처리 완료 순)
- 대기 중이 없는 모집글 섹션은 접힘 처리 (기본 상태)

---

## 상태 필터 탭

| 탭명 | 포함 상태 |
|------|----------|
| 대기 중 | 처리 전 신청 |
| 전체 | 대기 중 + 처리 완료 |

---

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 신청자 상세 보기 | 신청자 카드 탭 | DL3으로 이동 | — |
| 새로고침 | 당겨서 새로고침 | 목록 재조회 | — |

---

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 신청 1건 이상 | 섹션+카드 목록 표시 |
| 로딩 | 조회 중 | 스켈레톤 |
| 빈 상태 (대기 탭) | 처리 대기 신청 없음 | "처리할 신청이 없습니다" 안내 |
| 빈 상태 (전체 탭) | 신청 이력 없음 | "아직 신청이 없습니다" 안내 |
| 오류 | 네트워크 실패 | 오류 메시지 + 재시도 버튼 |

## 엣지케이스
- **48h 자동 거절 처리된 신청**: '거절' 배지로 표시, 카드는 '전체' 탭에서만 확인
- **DL1에서 특정 모집글 필터로 진입**: 해당 모집글 섹션만 펼쳐진 상태로 진입

- **신청자 카드**: DL3(신청자 상세) 헤더 카드와 동일 데이터 구조
