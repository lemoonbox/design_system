# CG1 — 상품 목록 (탐색)

## 개요
- 목적: 고객이 현재 모집 중인 시술 상품을 탐색하고 원하는 상품을 선택한다
- 진입 경로: 하단 탭 '탐색' / 로그인·가입 완료 후 자동 진입 (고객)
- 이탈 경로:
  - 상품 카드 탭 → CL1(상품 상세)

---

## 레이아웃 구조
- 네비게이션: 하단 탭 (탐색 활성)
- 헤더: 있음 — 서비스 로고 + 지역 표시 ("홍대")
- 주요 영역: 상품 카드 목록 (수직 스크롤)
- 하단 고정 영역: 없음 (하단 탭이 고정)

---

## 데이터 필드 (상품 카드 단위)

| 필드명 | 타입 | 필수 여부 | 표시 방식 | 비고 |
|--------|------|----------|----------|------|
| 참고 이미지 | image | N | 카드 상단 썸네일 (가로 전체) | 없으면 기본 플레이스홀더 |
| 시술명 | text | Y | 카드 본문 제목 | — |
| 디자이너명 | text | Y | 카드 본문 서브텍스트 | — |
| 시술 가격 | number | Y | 카드 본문 강조 텍스트 | 예: "5,000원" |
| 결제 방식 | text | Y | 배지 | "예약금 30%" 또는 "전액" |
| 가장 빠른 가능 일정 | date | Y | 카드 본문 | 예: "3/25 (수)" |
| 남은 모집 인원 | number | Y | 카드 하단 | 예: "2자리 남음" |
| 모집 상태 | select | Y | 배지 (모집 중 / 마감 임박 / 마감) | 마감된 상품은 카드 dimmed 처리 |

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 상품 상세 보기 | 카드 탭 | CL1으로 이동 | — |
| 새로고침 | 당겨서 새로고침 (pull-to-refresh) | 목록 재조회 | — |

---

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 상품 1개 이상 | 카드 목록 표시 |
| 로딩 | 첫 조회 또는 새로고침 중 | 카드 스켈레톤 3~4개 |
| 빈 상태 | 모집 중 상품 없음 | "현재 모집 중인 시술이 없습니다" 안내 + 재방문 유도 문구 |
| 오류 | 네트워크 실패 | 오류 메시지 + '다시 시도' 버튼 |

---

## 엣지케이스
- **마감된 상품 노출**: 마감 상품은 목록 하단으로 내려서 표시하고 카드 dimmed 처리; 탭 가능하나 CL1에서 신청 불가
- **모집 인원 실시간 변경**: 다른 고객이 신청해 인원이 줄어든 경우 CL1에서 최신 상태 확인 후 신청 처리

---

## 패턴 후보
- **상품 카드**: CL1(상품 상세) 헤더 요약과 동일 데이터 구조 — 공통 패턴화 대상
- **빈 상태 컴포넌트**: CG2(내 예약 빈 상태), DG1(모집글 없음)과 동일 레이아웃 구조
