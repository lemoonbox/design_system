# DL3 — 신청자 상세

## 개요
- 목적: 디자이너가 신청자의 프로필과 제출 사진을 열람하고 승인·거절·대안 일정 제안 중 하나를 선택하여 신청을 처리한다
- 진입 경로: DG2(신청 관리) → 신청자 항목 탭
- 이탈 경로:
  - 승인 또는 거절 처리 완료 → DG2(신청 관리)
  - 대안 일정 제안 → DM1(대안 일정 제안 Modal) 열기
  - DM1 발송 완료 → DG2(신청 관리)
  - 뒤로가기 → DG2(처리 없이 이탈)

---

## 레이아웃 구조
- 네비게이션: 상단 바 (뒤로가기 버튼 + 화면 제목 "신청자 정보")
- 헤더: 있음 — 모집글명 + 신청 접수 일시
- 주요 영역: 혼합 (신청자 정보 섹션 → 제출 사진 섹션 → 참고사항 섹션)
- 하단 고정 영역: 있음 — 액션 버튼 3종 (승인 / 거절 / 대안 일정 제안)

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 승인 | '승인' 버튼 탭 | 승인 확인 바텀시트 → 확인 탭 → 고객 알림 발송 → DG2로 이동 | 처리 대기 중 상태에서만 활성 |
| 거절 | '거절' 버튼 탭 | 거절 사유 선택 바텀시트 (사유 선택 필수) → 발송 → DG2로 이동 | 처리 대기 중 상태에서만 활성 |
| 대안 일정 제안 | '대안 일정 제안' 버튼 탭 | DM1(대안 일정 제안) Modal 열기 | 처리 대기 중 상태에서만 활성 |
| 사진 전체화면 보기 | 제출 사진 탭 | 전체화면 이미지 뷰어 오버레이 (핀치 줌 지원) | — |
| 뒤로가기 | 상단 바 뒤로가기 탭 | DG2로 이동 | 처리 완료 여부 무관 |

### 거절 사유 목록 (바텀시트)
- 사진이 기준에 맞지 않습니다
- 해당 일정이 불가합니다
- 모집 인원이 마감되었습니다
- 기타 (직접 입력 불가 — MVP 단순화)

## 엣지케이스
- **OI-08 사진 열람 권한**: 제출 사진은 해당 신청을 받은 디자이너만 열람 가능; 다른 디자이너·고객 직접 URL 접근 시 403
- **신청자 탈퇴 처리**: 신청자가 계정을 탈퇴한 경우 이름 "탈퇴한 사용자", 사진 자리 표시자로 처리
- **이미 처리된 신청 재진입**: DG2에서 완료된 항목 탭 시 읽기 전용 상태로 진입 (처리 버튼 비활성)
- **48h 자동 거절 중 수동 처리 시도**: 자동 거절과 동시 처리 시 409 응답 → "이미 처리된 신청입니다" 안내
- **모집 인원 마감 후 남은 대기 신청자**: 마감 배지 + 버튼은 여전히 활성 (디자이너가 초과 승인 방지는 서버 검증)

## 패턴 후보
- **전체화면 이미지 뷰어**: CL2(신청하기) 사진 업로드 확인 시 동일 뷰어 재사용 가능
- **처리 확인 바텀시트**: 승인 확인·거절 사유 바텀시트 — DM2(시술 완료 확인)의 확인 구조와 유사, 공통 패턴 후보
- **상태 배지**: DG2(신청 관리) 목록과 동일한 상태 배지 시스템 공유
