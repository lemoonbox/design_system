# A2 — 로그인

## 개요
- 목적: 휴대폰 번호와 인증번호로 본인 확인 후 서비스에 로그인한다
- 진입 경로: A1 '로그인' 버튼 / 비로그인 상태에서 보호 화면 접근 시 리다이렉트
- 이탈 경로:
  - 로그인 완료 → 역할에 따라 CG1(고객) 또는 DG1(디자이너)
  - 로그인 완료 (리다이렉트된 경우) → 리다이렉트 전 화면으로 복귀
  - '회원가입' 링크 탭 → A3

---

## 레이아웃 구조
- 네비게이션: 상단 바 (뒤로가기 버튼 → A1)
- 헤더: 있음 — "로그인" 타이틀
- 주요 영역: 폼 (휴대폰 번호 입력 → 인증번호 발송 → 인증번호 입력 단계)
- 하단 고정 영역: 있음 — 단계별 CTA 버튼 ('인증번호 발송' → '로그인')

---

## 데이터 필드

| 필드명 | 타입 | 필수 여부 | 표시 방식 | 비고 |
|--------|------|----------|----------|------|
| 휴대폰 번호 | text (숫자) | Y | 입력 필드 + '발송' 버튼 | 010-XXXX-XXXX 형식, 자동 하이픈 삽입 |
| 인증번호 | text (숫자 6자리) | Y | 입력 필드 + 카운트다운 타이머 | 인증번호 발송 후 필드 표시; 유효시간 3분 |
| 재발송 링크 | — | — | 타이머 하단 텍스트 링크 | 60초 후 활성화 |

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 인증번호 발송 | '발송' 버튼 탭 | SMS 발송 + 인증번호 입력 필드 표시 + 3분 타이머 시작 | 휴대폰 번호 형식 유효 |
| 인증번호 발송 실패 | '발송' 버튼 탭 | 인라인 오류 메시지 | 형식 오류 또는 SMS 발송 실패 |
| 로그인 | '로그인' 버튼 탭 | 토큰 발급 → 역할 확인 → CG1/DG1 이동 | 인증번호 일치 |
| 로그인 실패 | '로그인' 버튼 탭 | 인라인 오류 ("인증번호가 올바르지 않습니다") | 인증번호 불일치 또는 만료 |
| 인증번호 재발송 | '재발송' 링크 탭 | SMS 재발송 + 타이머 리셋 | 60초 경과 후 |
| 회원가입 이동 | '회원가입' 텍스트 링크 탭 | A3으로 이동 | — |
| 뒤로가기 | 상단 바 뒤로가기 탭 | A1으로 이동 | — |

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 번호 입력 | 초기 진입 | 휴대폰 번호 필드만 표시, '발송' 버튼 비활성 |
| 번호 입력 완료 | 번호 11자리 입력 | '발송' 버튼 활성 |
| 인증번호 대기 | 발송 성공 | 인증번호 필드 + 타이머 표시, '로그인' 버튼 비활성 |
| 인증번호 입력 완료 | 6자리 입력 | '로그인' 버튼 활성 |
| 타이머 만료 | 3분 경과 | 인증번호 필드 비활성, "인증 시간이 만료되었습니다" + 재발송 안내 |
| 로그인 처리 중 | API 호출 중 | CTA 버튼 로딩 |
| 오류 | 인증 실패 | 인라인 오류 표시 |

## 엣지케이스
- **미가입 번호로 로그인 시도**: 서버 404 응답 → "가입되지 않은 번호입니다. 회원가입 하시겠습니까?" 안내 + A3 이동 링크
- **SMS 발송 실패 (통신사 오류)**: 재시도 안내 + 고객센터 링크 (P1)
- **인증번호 5회 연속 실패**: 10분 잠금 안내
- **이미 로그인된 상태로 접근**: A1 자동 로그인 로직에서 처리, A2 미진입
