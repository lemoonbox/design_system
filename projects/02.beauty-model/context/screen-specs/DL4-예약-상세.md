# DL4 — 예약 상세 (디자이너)

## 개요
- 목적: 디자이너가 확정 예약의 상세 정보를 확인하고 시술 완료를 처리한다
- 진입 경로: DG3(예약 목록) → 예약 카드 탭
- 이탈 경로:
  - '시술 완료' 버튼 탭 → DM2(시술 완료 확인 Modal)
  - DM2 완료 확인 → DG3(완료 상태)
  - 뒤로가기 → DG3

---

## 레이아웃 구조
- 네비게이션: 상단 바 (뒤로가기 + "예약 상세" 타이틀)
- 헤더: 있음 — 예약 상태 배지 (대형)
- 주요 영역: 스크롤 (고객 정보 → 시술 정보 → 결제 정보)
- 하단 고정 영역: 있음 — 상태별 CTA 버튼

---

## 상태별 하단 CTA

| 예약 상태 | 하단 CTA |
|----------|---------|
| 확정 | '시술 완료' 버튼 (Primary) |
| 완료 | 없음 ("시술이 완료된 예약입니다" + 정산 예정일 표시) |
| 취소 | 없음 ("취소된 예약입니다" 안내) |

## 엣지케이스
- **고객이 탈퇴한 경우**: 고객명 "탈퇴한 사용자", 연락처 미표시
- **노쇼 처리 (예약 당일 미방문)**: MVP에서는 디자이너가 수동으로 '시술 완료'를 누르지 않으면 완료 미처리; 운영자가 노쇼 기록 처리 (P1 자동화)
