# CL2 — 신청하기

## 개요
- 목적: 고객이 희망 일정을 선택하고 사진을 제출하며 동의 항목을 확인한 뒤 시술 신청을 완료한다
- 진입 경로: CL1(상품 상세) → '신청하기' 버튼 탭 (로그인 필수; 비로그인 시 A2로 리다이렉트)
- 이탈 경로:
  - 신청 완료 → CG2(내 예약, 대기 중 상태)
  - 뒤로가기 → CL1(상품 상세)

---

## 레이아웃 구조
- 네비게이션: 상단 바 (뒤로가기 버튼 + 화면 제목 "신청하기")
- 헤더: 있음 — 시술명, 디자이너명, 시술 가격 요약 카드 (상단 고정)
- 주요 영역: 폼 (일정 선택 → 사진 제출 → 동의 체크 순서 고정)
- 하단 고정 영역: 있음 — '신청하기' CTA 버튼 (필수 필드 미완료 시 비활성)

## 액션

| 액션명 | 트리거 | 결과 | 조건 |
|--------|--------|------|------|
| 신청 완료 | '신청하기' 버튼 탭 | CG2(내 예약, 대기 중)로 이동 | 희망 일정·사진·동의 체크 모두 완료 |
| 신청 실패 (검증) | '신청하기' 버튼 탭 | 미완료 필드에 인라인 오류 표시, 화면 유지 | 필수 필드 하나라도 미완료 |
| 내용 보기 | 동의 항목 옆 '내용 보기' 링크 탭 | CM2(동의 내역 확인) Modal 열기 | — |

## 상태 정의

| 상태 | 발생 조건 | UI 처리 |
|------|----------|---------|
| 기본 | 화면 진입 직후 | 헤더 카드 표시, 폼 빈 상태, CTA 비활성 |
| 입력 중 | 사용자가 필드 채우는 중 | 각 필드 포커스/입력 상태 UI |
| 제출 가능 | 필수 필드 전체 완료 | CTA 버튼 활성화 |
| 제출 중 | API 호출·사진 업로드 진행 중 | CTA 버튼 로딩 스피너, 폼 인터랙션 비활성 |
| 슬롯 없음 | 선택 가능한 일정 슬롯이 모두 마감 | 일정 선택 영역에 "현재 신청 가능한 일정이 없습니다" 안내, CTA 비활성 |
| 오류 | 네트워크 실패 또는 업로드 오류 | 토스트 오류 메시지 + 재시도 유도 |

## 엣지케이스
- **노쇼 3회 초과 고객**: 화면 진입 시 신청 제한 안내 배너 표시 + CTA 비활성 (P1 — 운영자 플래그 기반)
- **사진 용량 초과**: 10MB 초과 파일 선택 시 즉시 오류 메시지, 해당 파일 첨부 차단
- **세션 만료 상태로 제출 시도**: 제출 API 401 응답 → A2(로그인)으로 리다이렉트 후 복귀 처리
- **디자이너가 모집글을 신청 직전 마감·삭제**: 제출 시 409 응답 → "모집이 마감되었습니다" 안내 후 CG1으로 이동
- **동일 상품·동일 슬롯 중복 신청 시도**: 제출 시 409 응답 → "이미 신청한 일정입니다" 인라인 오류

## 패턴 후보
- **사진 업로드 그리드**: CL6(내 사진 관리)의 사진 그리드와 동일 구조 — 공통 패턴화 대상
- **동의 체크 블록**: DM2(시술 완료 확인)에서 동의 내역 재확인 시 동일 체크박스 UI 재사용
- **헤더 요약 카드**: CL3(결제), CL4(예약 상세) 상단에도 동일한 시술 요약 카드 반복 — 공통 패턴화 대상
