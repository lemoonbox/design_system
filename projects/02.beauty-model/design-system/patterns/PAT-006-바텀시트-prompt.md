# PAT-006 — 바텀 시트

## 메타
- 등장 화면: CM1 (취소 확인), CM2 (동의 내역 확인), DM1 (대안 일정 제안), DM2 (시술 완료 확인)
- 역할: 확인·취소·동의 등 사용자의 명시적 의사 결정이 필요한 액션을 화면 하단에서 모달 방식으로 요청한다. 배경 딤 처리와 함께 사용되며, 사용자가 반드시 응답해야 하는 차단형 인터랙션이다
- 반복 구조: CM1·CM2·DM1·DM2 네 화면 모두 화면 하단에서 올라오는 동일한 시트 구조를 사용

## 시각적 의도

> 이 패턴은 **화면 하단에서 올라온 흰색 시트**처럼 보인다. 사용자의 시선은 상단 드래그 핸들 → 제목 → 설명 텍스트 → 하단 액션 버튼 순으로 자연스럽게 흐른다.

- **전체 인상**: 배경 딤(#000000 30% 오버레이) 위에 white surface 시트가 하단 radius.2xl과 shadow-md를 가지고 떠오른 모습. 서비스 내에서 유일하게 shadow를 쓰는 레이어
- **시각적 초점**: 하단 주요 CTA 버튼 (brand.600 · pill) — 사용자가 취해야 할 액션
- **시각적 무게**: 텍스트 콘텐츠 영역 60% : 버튼 영역 40%
- **디자인 핵심**: 상단 radius.2xl(16px)의 부드러운 곡선 + shadow-md의 은은한 깊이감이 시트를 콘텐츠 위에 떠 있는 레이어로 명확히 인식하게 만든다

---

## Visual Recipe

반투명 딤 위로 흰 시트가 하단에서 올라오며, 상단 모서리만 16px 둥글고 shadow-md로만 깊이가 생긴다(테두리 없음). 맨 위 중앙의 짧은 핸들이 “드래그 가능한 시트”라는 관습을 살짝 보여 주고, 제목(subheading)→설명(body · secondary)이 차례로 읽힌 뒤 시선은 곧바로 너비 꽉 찬 보라색 CTA로 떨어진다. 보조 버튼은 흰 배경에 회색 테두리로 한 단 낮춰, 파괴적이거나 되돌리기 어려운 액션과 보조 액션의 위계가 색과 면으로 분명해진다. 서비스 전역에서 거의 유일하게 elevation을 쓰는 순간이라 집중도가 높다.

## 레이아웃 구조

### 전체 프레임 `바텀-시트`

| 속성 | 값 |
|------|-----|
| 크기 | 390 × auto (최소 200px, 내용에 따라 가변) |
| 배경 | surface — white #ffffff |
| 배치 | 수직 스택 |
| 주축 정렬 | 시작 |
| 교차축 정렬 | 시작 |
| 내부 여백 | 상20 하32 좌16 우16 — 상: 20px, 하: 32px (safe area 포함), 좌우: spacing.xl |
| 요소 간격 | 20px |
| 모서리 | 상단만 16px — radius.2xl (상단 좌우 라운드, 하단은 각짐) |
| 깊이 | shadow-md — x:0 y:4 blur:8 spread:-2 color:#0a0d12 alpha:0.10 |
| 테두리 | 없음 (shadow로 구분, 동시 사용 금지) |

### 구조 다이어그램

```
┌─────────────────────────────────────────────────────┐ ← 상단 radius.2xl 16px
│              ┌──────┐                               │ ← 드래그 핸들 (중앙)
│              └──────┘                               │   40×4px · gray.300
│                                                     │
│  제목 텍스트 (subheading)                              │
│  설명 텍스트 (body · text-secondary)                   │
│                                                     │
│  ┌─────────────────────────────────────────────┐   │
│  │  주요 CTA 버튼 (brand.600 · full-width · pill)│   │
│  └─────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────┐   │
│  │  보조 버튼 (border·white bg · full-width · pill)│  │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘ ← 하단 각짐
```

## 시각 품질 검증

### Surface Depth
| 요소 | fill | 부모/배경 fill | 대비 | 조치 |
|------|------|--------------|------|------|
| 시트 전체 | surface #ffffff | 딤 오버레이 (#000000 30%) | 충분 (white vs 어두운 딤) | shadow-md 추가로 깊이 보강 |
| CTA 버튼 | primary #7f56d9 | surface #ffffff | 충분 (보라 vs 흰색) | — |
| 보조 버튼 | white #ffffff | surface #ffffff | 동일 → border로 구분 | border gray.300 1px 필수 |

### Elevation
| 요소 | 유형 | shadow 토큰 | 근거 |
|------|------|------------|------|
| 시트 전체 | 모달/오버레이 | shadow-md (x:0 y:4 blur:8 spread:-2 #0a0d12 10%) | visual-direction: "예외적으로 바텀시트·모달만 shadow-md 적용" |

### 타이포 위계
| 수준 | 역할 | 스펙 (크기/행간/자간/굵기) | 적용 요소 |
|------|------|-------------------------|----------|
| 1 (최강) | subheading | 18px/28px/0px/600 | 시트 제목 |
| 2 | body | 16px/24px/0px/400 | 설명 텍스트 |
| 3 | button | 14px/20px/0px/600 | 버튼 텍스트 |

### 강조
- focal point: 주요 CTA 버튼 — brand.600 fill + white 텍스트 + pill radius의 시각적 무게가 가장 큼
- 상태 표현: 버튼 계층(primary vs secondary) + 텍스트로 액션 우선순위 명시
- 근거: visual-direction.mdc > CTA 버튼 (brand.600 fill + white 텍스트, radius.full)

### 밀도
- CTA 버튼 터치 타겟: 48px — 44px 충족 ✅
- 보조 버튼 터치 타겟: 48px — 44px 충족 ✅
- 드래그 핸들: 시각적 인디케이터. 터치 타겟은 핸들 주변 20px 여백 포함으로 사실상 40×40 이상 확보
