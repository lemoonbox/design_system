# PAT-003 — 신청자 행

## 메타
- 등장 화면: DG2 (신청 관리 — 디자이너), DL3 (신청자 상세 — 디자이너)
- 역할: 시술 신청자 1명의 프로필 사진·이름·신청 상태를 한 행에 표시하여 디자이너가 신청자 목록을 빠르게 파악하고 탭으로 진입할 수 있게 한다
- 반복 구조: DG2·DL3 두 화면 모두 신청자 목록을 세로 행 리스트로 표시하며, 이 패턴이 반복 단위로 쌓인다

## 시각적 의도

> 이 패턴은 **프로필 원형 아바타 + 이름 + 상태 배지가 한 줄에 나란히 있는 연락처 행**처럼 보인다. 사용자의 시선은 먼저 좌측 아바타 → 이름 → 우측 상태 배지 순으로 흐른다.

- **전체 인상**: 인물 중심의 컴팩트 행. 원형 아바타가 시각적 앵커 역할을 하고, 이름과 상태 배지가 좌우로 대비된다
- **시각적 초점**: 좌측 원형 아바타 + 이름 (body-medium · text-primary)
- **시각적 무게**: 아바타+이름 블록 70% : 상태 배지+화살표 30%
- **디자인 핵심**: 40×40px 원형 아바타의 gray.100 배경과 상태 배지의 tint 색상이 행 안에서 두 개의 색상 포인트를 만들어 정보를 명확하게 분리한다

---

## Visual Recipe

왼쪽 원형 아바타가 연락처 리스트처럼 인물을 먼저 인식시키고, 바로 옆 세로 스택에는 이름(body-medium)이 굵게, 그 아래 신청일(caption)이 한 톤 낮춰 붙는다. 오른쪽에는 PAT-004 배지가 신청 처리 상태를 색으로 말하고, chevron이 상세 화면 진입을 암시한다. 행 높이는 비교적 낮게 유지되지만 전체 행이 탭 타겟이므로 터치는 안정적이다. 디자이너 업무 맥락에 맞게 감정 과장 없이, 사람·상태·다음 행동만 빠르게 읽히는 효율형 UI로 보인다.

### 강조
- focal point: 좌측 원형 아바타 — 원형 형태가 직사각형 텍스트 요소들 사이에서 시선을 먼저 끌어당김
- 이차 강조: 우측 상태 배지의 tint 색상 — 신청 처리 여부 판단의 핵심 정보
- 근거: visual-direction.mdc > 강조 방식 (상태 배지: 상태별 tint 배경 + 상태 텍스트 색)

### 밀도
- 터치 가능 요소: 전체 행 — 높이 64px — 44px 충족 ✅
- 아바타 자체 터치 타겟: 40px — 최소 44px에 약간 미달. 행 전체가 탭 영역이므로 실질적 문제 없음
