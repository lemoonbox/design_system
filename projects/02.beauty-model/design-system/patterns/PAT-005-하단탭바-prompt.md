# PAT-005 — 하단 탭바

## 메타
- 등장 화면: CG1, CG2, CG3, CG4 (고객 탭), DG1, DG2, DG3, DG4 (디자이너 탭)
- 역할: 고객·디자이너 각 역할의 주요 섹션을 전환하는 고정 하단 내비게이션이다. 현재 위치를 항상 명시하고 핵심 섹션 간 이동을 1탭으로 가능하게 한다
- 반복 구조: CG1~4, DG1~4 총 8개 화면의 최하단에 항상 고정 표시. 역할(고객/디자이너)에 따라 탭 구성이 달라지는 2가지 변형이 존재

## 시각적 의도

> 이 패턴은 **4개의 균등 칸으로 나뉜 흰색 하단 바**처럼 보인다. 사용자의 시선은 활성 탭 아이콘(brand.600 보라색)에 먼저 끌리고, 이후 탭 레이블 텍스트를 읽는다.

- **전체 인상**: 깔끔하고 대칭적인 하단 탭. 활성 탭만 brand.600 보라색으로 강조되고 나머지는 gray.400으로 후퇴
- **시각적 초점**: 현재 활성 탭 — 아이콘 + 레이블 모두 brand.600으로 강조
- **시각적 무게**: 4개 탭 각 25%. 활성 탭만 색상 무게가 높음
- **디자인 핵심**: 활성/비활성의 명확한 색상 대비(brand.600 vs gray.400)가 현재 위치를 즉각 인식하게 만든다. 상단 border(gray.300) 1px이 콘텐츠 영역과 탭바를 구분한다

---

## Visual Recipe

화면 하단에 흰 바가 고정되고, 위쪽 가장자리만 gray.300 1px 선으로 본문과 나뉜다. 네 칸이 균등하게 쪼개져 각각 아이콘(24px)과 짧은 레이블(label)이 세로로 중앙 정렬된다. 현재 탭 한 곳만 아이콘·텍스트가 동시에 brand.600으로 올라오고 나머지 세 칸은 gray.400으로 눌려 있어, 역할 전환 없이도 “지금 어디 탭인지”가 즉시 읽힌다. 그림자는 쓰지 않아 콘텐츠와 동일한 플랫 계층 안에 머무르며, 모바일 웹에서도 과하지 않은 전문형 내비게이션 인상을 준다.

## 레이아웃 구조

### 전체 프레임 `하단-탭바`

| 속성 | 값 |
|------|-----|
| 크기 | 390 × 64 px (safe area 제외 순수 탭바 높이) |
| 배경 | surface — white #ffffff |
| 배치 | 수평 스택 |
| 주축 정렬 | 양끝 (space-between 또는 각 탭 25% 균등 배분) |
| 교차축 정렬 | 중앙 |
| 내부 여백 | 상0 하0 좌0 우0 |
| 요소 간격 | 0 (4탭이 390px를 균등 분할) |
| 모서리 | 없음 |
| 깊이 | 없음 |
| 테두리 | 상단만 border — gray.300 #d5d7da 1px |

> iOS safe area를 위한 추가 높이(34px)는 safe area 프레임을 별도로 탭바 하단에 추가한다. 해당 영역: 390×34px, 배경 white, 콘텐츠 없음.

### 구조 다이어그램

```
 상단 border 1px (gray.300)
┌──────────┬──────────┬──────────┬──────────┐  390×64
│  탭-1    │  탭-2    │  탭-3    │  탭-4    │
│  [아이콘] │  [아이콘] │  [아이콘] │  [아이콘] │  4개 균등 분할 (97.5px)
│  레이블  │  레이블  │  레이블  │  레이블  │
└──────────┴──────────┴──────────┴──────────┘
```

## 시각 품질 검증

### Surface Depth
| 요소 | fill | 부모/배경 fill | 대비 | 조치 |
|------|------|--------------|------|------|
| 탭바 전체 | surface #ffffff | 화면 콘텐츠 (다양) | 충분 | 상단 border gray.300 1px로 명확 구분 |

### Elevation
| 요소 | 유형 | shadow 토큰 | 근거 |
|------|------|------------|------|
| 탭바 | 하단 고정 내비 | 없음 | visual-direction: 플랫 스타일. 바텀시트·모달만 shadow-md. 탭바는 border로 구분 |

### 타이포 위계
| 수준 | 역할 | 스펙 (크기/행간/자간/굵기) | 적용 요소 |
|------|------|-------------------------|----------|
| 단일 | label | 14px/20px/0px/500 | 탭 레이블 (활성·비활성 동일 스펙, 색상으로만 구분) |

### 강조
- focal point: 활성 탭 — 아이콘 + 레이블 brand.600 #7f56d9 (4탭 중 하나만 강조)
- 상태 표현: 활성 = brand.600 (아이콘 + 레이블 동시 색상 변경), 비활성 = gray.400
- 근거: visual-direction.mdc > 브랜드 방향 (Primary: brand.600 — 활성 탭)

### 밀도
- 탭 아이템 터치 타겟: 97×64px — 44px 충족 ✅ (너비·높이 모두 충분)
- 탭 간 구분: 시각적 구분선 없음. 4등분 균등 배치로 자연스러운 탭 영역 인식
