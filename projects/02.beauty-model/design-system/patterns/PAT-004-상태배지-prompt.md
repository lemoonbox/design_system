# PAT-004 — 상태 배지

## 메타
- 등장 화면: CG2, CL4, DG2, DG3, DL3, DL4 (전체 6개 화면)
- 역할: 예약·신청·시술의 처리 상태(대기/승인/거절/완료)를 한눈에 전달하는 소형 상태 칩이다. PAT-001·PAT-003에서 인라인으로 사용되며, 상세 화면 헤더에도 독립적으로 등장한다
- 반복 구조: 6개 화면 공통 상태 표현 단위. 항상 tint 배경 + 상태 텍스트 색 조합으로 표현

## 시각적 의도

> 이 패턴은 **작고 둥근 색상 칩**처럼 보인다. 사용자의 시선은 배지 배경색에 먼저 반응하고, 이어 짧은 상태 텍스트를 읽는다.

- **전체 인상**: 최소한의 크기로 최대한 명확한 상태를 전달하는 pill 형태의 칩. 배경 tint + 텍스트 색 조합으로 색상만으로 상태를 구분하지 않는 원칙을 지킨다
- **시각적 초점**: 배경 tint 색상 → 텍스트 문구 순서로 인식
- **시각적 무게**: 색상 배경 50% : 텍스트 50%
- **디자인 핵심**: pill radius(radius.full)의 부드러운 라운드와 상태별 tint 색상이 서로 다른 상태를 직관적으로 구분하게 만든다. 색상 + 문구를 병행해 접근성을 확보한다

---

## Visual Recipe

높이 24px짜리 작은 알약 형태의 칩이 흰 카드나 리스트 행 위에 떠 있지 않고 자연스럽게 붙어 있다. 배경은 상태별 연한 tint(warning.50 / success.50 / error.50 / gray.100), 글자는 같은 계열의 진한 상태색으로 맞춰 “색만 보고도 대략 알 수 있지만, 문구로 확정”되는 구조다. 상세 화면 변형에서는 14px 아이콘을 텍스트 왼쪽에 얹어 시각적 무게를 조금 올릴 수 있다. 전체 서비스 톤(차분·신뢰)을 해치지 않으면서도 예약·신청 상태처럼 시간 민감한 정보는 한눈에 걸리도록 설계된다.

## 레이아웃 구조

### 전체 프레임 `상태-배지`

| 속성 | 값 |
|------|-----|
| 크기 | auto × 24 px (내용에 따라 너비 자동) |
| 배치 | 수평 스택 |
| 주축 정렬 | 중앙 |
| 교차축 정렬 | 중앙 |
| 내부 여백 | 상3 하3 좌8 우8 |
| 요소 간격 | 4px (아이콘 있을 때) |
| 모서리 | radius.full — 완전한 pill (높이 24px → radius 12px) |
| 깊이 | 없음 |
| 테두리 | 없음 |

### 구조 다이어그램 (텍스트 전용 기본형)

```
┌────────────────┐  auto×24
│  [상태 텍스트]   │  수평 중앙, pill
└────────────────┘
```

### 구조 다이어그램 (아이콘 + 텍스트 변형)

```
┌──────────────────────┐  auto×24
│  [🔵 아이콘]  [텍스트] │  수평, gap 4px, pill
└──────────────────────┘
```

### 강조
- focal point: 배경 tint 색상 — 색상만으로 상태를 직관적으로 구분
- 상태 표현: 배경 tint + 텍스트 문구 + 아이콘(선택) 3중 표현으로 색각 접근성 보완
- 근거: visual-direction.mdc > 상태 표현 (tint 배경 + 상태 텍스트 색), 확정 디자인 결정 > 상태 표현

### 밀도
- 배지 자체 터치 타겟: 높이 24px — 단독 버튼이 아니므로 44px 규칙 미적용 (항상 더 큰 행·카드 내 포함됨)
- 독립 탭 화면(CL4·DL4)에서 단독 표시 시: 주변 여백을 포함하여 터치 영역 확보 권장
