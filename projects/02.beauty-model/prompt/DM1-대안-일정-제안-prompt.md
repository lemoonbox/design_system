# 디자인 명세 — DM1 대안 일정 제안

## 화면 개요
- **목적**: 신청자 희망 일정이 맞지 않을 때, 모집글 **잔여 슬롯 중 하나**를 골라 고객에게 대안 일정을 제안한다.
- **디바이스**: 390 × 844 모바일 (Modal 오버레이)
- **사용자 맥락**: DL3 신청자 상세에서 ‘대안 일정 제안’을 탭한 직후
- **핵심 행동**: 슬롯 선택 후 **제안 발송**

## 시각적 의도

- **분위기**: 짧고 명확한 **바텀 시트형** 결정 화면. 배경은 딤 처리되고, 시트만 읽기 쉬운 surface+elevation.
- **시각적 초점 (focal point)**: **라디오 목록으로 펼쳐진 잔여 슬롯**과 하단 **제안 발송** CTA.
- **색상 톤**: 시트 surface 흰색, 참고 안내는 brand.50 또는 gray.100 박스로 희망 일정을 분리 표시.
- **밀도**: 조밀 — 슬롯 행 높이 최소 48px, 행 간 border-subtle.
- **디자인 핵심**: visual-direction에 따라 시트에만 **shadow-md**; 카드 내부는 border 중심.

---

## Visual Recipe

화면 전체를 반투명 딤으로 덮고, 하단에서 올라온 둥근 상단 모서리의 흰 시트가 **PAT-006**의 무게감(shadow-md)을 갖는다. 헤더는 제목+신청자명이 한 덩어리로 보이고, 우측 상단 **X**로 닫기가 분명하다. 그 아래 **고객 희망 일정**은 작은 정보 박스로 고정 표시한 뒤, **선택 가능한 슬롯**은 라디오+날짜시간 텍스트가 한 줄에 정렬되어 빠르게 비교된다. 슬롯이 없을 때는 빈 목록 대신 **아이콘+안내 문구**로 면을 채운다.

## 레이아웃 구조

### 전체 구조

- **딤(스크림)**: gray.900 #181d27, opacity 약 40% — rgba(24,29,39,0.4)
- **시트**: 하단 고정, 좌우 하단 코너 radius.2xl(16px), 상단 좌우만 radius.2xl (바텀시트 관례)
- **시트 깊이**: **shadow-md** — x:0 y:4 blur:8 spread:-2 #0a0d12 10%

```
        ┌─────────────────────┐
        │ 딤 (전체 탭 닫기)    │
        │                     │
        │   ╭─────────────────╮
        │   │ 헤더  제목  [x] │
        │   │ 참고: 희망 일정  │
        │   │ ─────────────── │
        │   │ ○ 슬롯 1        │
        │   │ ○ 슬롯 2        │
        │   │ ...             │
        │   │ [제안 발송]      │
        │   ╰─────────────────╯
```

## 시각 품질 검증

### Surface Depth
| 딤 | 반투명 gray.900 | 시트 white | 대비 충분 |
| 참고 박스 | brand.50 | white 목록 | 인접 대비 충분 |

### Elevation
| 시트 전체 | 바텀시트 | **shadow-md** | visual-direction 예외 규정 |
| 목록 행 | 플랫 | 없음 | border-subtle |

### 타이포 위계
| heading(제목) / body(슬롯) / caption(부가) — 3단 이내 |

### 강조 검증
- focal: 라디오 블록 + CTA
- primary: 선택된 라디오, CTA, 참고 박스 아이콘

### 밀도 검증
- 행 48px, 닫기·CTA 터치 44px 이상

### 시각 밀도 검증
- 슬롯 3개 이상 시 좌측 **얇은 brand.3px 바**를 선택 행에만 적용해 반복 행 차별화 가능
- 빈 상태는 아이콘+2줄 텍스트로 **빈 면 최소화**
- 헤더에 **좌측 3px brand 바** 옵션으로 섹션 무게 보강 가능
