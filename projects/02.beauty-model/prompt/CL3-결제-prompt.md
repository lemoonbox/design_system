# 디자인 명세 — CL3 결제

## 화면 개요
- **목적**: 디자이너 승인 후 예약금 또는 전액을 결제해 예약을 확정한다.
- **디바이스**: 390 × 844 모바일
- **사용자 맥락**: 예약 상세(CL4)에서 ‘결제하기’로 진입했을 때, 시간 제한(24h 등)을 인지한 채 금액을 확인하고 PG로 넘긴다.
- **핵심 행동**: 결제 수단을 확인·선택한 뒤 하단 ‘결제하기’로 PG를 호출한다.

## 시각적 의도

- **분위기**: 신뢰·안심이 느껴지는 결제 전용 화면. 상단 요약으로 맥락을 고정하고, 중앙은 금액·수단이 한눈에 들어오게 정돈한다.
- **시각적 초점 (focal point)**: **결제 금액(대형 숫자) + 하단 결제 CTA** — 금액이 숫자 위계 최상단, 버튼이 행동 종착점.
- **색상 톤**: 배경 gray.50, 카드 white+border. 금액 강조는 text-primary + 필요 시 금액 블록만 **primary-surface brand.50** 틴트. 마감 임박은 **warning** tint 배너로 시간 압박을 전달.
- **밀도**: 조밀 — 행 구분은 border-subtle, 불필요한 카드 래핑 최소화.
- **디자인 핵심**: 24h 카운트다운 배너의 가시성, 금액 상세 행의 스캔 가능성, 결제 수단 행에 브랜드 결제 아이콘(토큰)으로 빈 슬롯 없이 전문감을 준다.

## Visual Recipe

전체 배경 **gray.50** 위에 상단 바와 **시술·확정 일정 요약 카드(흰 surface, 1px gray.300, radius.2xl)** 를 둔다. 그 아래 **24h 만료 안내 배너**가 있으면 **warning.50 배경 + clock 아이콘**에 타이틀은 **warning — warning.500 #f79009 (rgba(247,144,9,1))**, 보조 문구는 **caption · text-secondary**로 두어 tint+텍스트색을 병행한다. **‘결제 금액’ 블록**은 **brand.50 틴트 카드** 안에 라벨은 **label**, 금액 숫자는 역할 **heading(20px/30px/0 Semibold)** 으로 두어 design-system 타이포만 쓰면서도 블록 내 최대 위계로 읽힌다. 그 아래 **시술비·결제 방식** 등은 **body** 행으로 나열하고 행 사이는 **border-subtle**로 구분한다. 결제 수단에는 **payment 토큰(md) 아이콘**과 선택 상태를 보여 주며, **shield + caption**으로 수수료 안내를 붙인다. 하단 고정 바에는 **"{금액}원 결제하기"** 라벨의 **primary pill** 버튼을 둔다.

### 강조 검증
- focal point: **금액 숫자 + CTA** — brand.50 블록으로 금액 묶음
- primary 사용: CTA, 섹션 바, (선택) 결제 수단 선택 테두리
- 상태 표현: 만료 임박은 **아이콘 + tint + 문구**

### 밀도 검증
- 터치: 뒤로, 규정 헤더, 수단 행, CTA **≥44px**
- ❌ 미충족 시: 행 높이·패딩 증가

### 시각 밀도 검증
- 스크롤 영역: 요약·배너·틴트 금액 블록·행 리스트·수단으로 **빈 단색면 50% 미만**
- 텍스트 전용 카드 없음 — **아이콘·행·배너**로 앵커
- 섹션 헤더: **4px 바 + 제목**
- 네비: 상단 바만 — **56px 명확한 타이포**
